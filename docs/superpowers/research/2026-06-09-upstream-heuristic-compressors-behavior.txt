SUBSYSTEM: reformats/json_minifier   ->   internal/reformats   fetchedOk= True
====================================================================================================
appliesTo: ['json_array']

estimateBloat: n/a — no EstimateBloat. The minifier performs a full serde_json parse + compact re-emit unconditionally (no cheap pre-pass / structural sniff). The only "cheap" early check is trimmed.is_empty() before parsing.
confidence: n/a — reformats are not offload transforms; there is no Confidence() function, no EstimateBloat sniff, no bloat heuristic in this path.
ccrMarker: n/a — reformat transforms never involve CCR or marker emission. No min_compression_ratio_for_ccr gate exists in this subsystem; the only gate is the never-inflate guard (output_len >= input_len → return original). ReformatOutput carries no marker field; it is purely {output: String, bytes_saved: usize}.

-- publicApi --
  * JsonMinifier: pub struct JsonMinifier; (derive Debug, Default, Clone, Copy) — zero-sized unit struct
      Whitespace-stripping JSON minifier implementing ReformatTransform. Handles both JSON arrays and objects (detector folds both into the JsonArray content type).
  * JsonMinifier::name: fn name(&self) -> &'static str
      Returns the literal constant NAME = "json_minifier".
  * JsonMinifier::applies_to: fn applies_to(&self) -> &[ContentType]
      Returns &[ContentType::JsonArray] — a one-element slice. The detector uses JsonArray as the umbrella tag for all structurally-recognized JSON (arrays AND objects); the minifier doesn't care about top-level shape.
  * JsonMinifier::apply: fn apply(&self, content: &str) -> Result<ReformatOutput, TransformError>
      Core transform: trim, parse to serde_json::Value, re-emit compact, never-inflate guard, return ReformatOutput{output, bytes_saved}.
  * ReformatOutput: pub struct ReformatOutput { pub output: String, pub bytes_saved: usize } — derive Debug, Clone
      Result of a reformat. Only two fields: the (possibly minified) output string and how many bytes were saved. No marker, no original_len field, no CCR fields.
  * ReformatOutput::from_lengths: pub fn from_lengths(input_len: usize, output: String) -> Self
      Constructor computing bytes_saved = input_len.saturating_sub(output.len()), then storing output. Note: input_len is the FIRST positional arg (an explicit length passed by the caller), output is moved in second.
  * ReformatTransform (trait): pub trait ReformatTransform: Send + Sync { fn name(&self)->&'static str; fn applies_to(&self)->&[ContentType]; fn apply(&self,content:&str)->Result<ReformatOutput,TransformError>; }
      The reformat-transform contract. Output bytes are semantically equivalent to input (LLM gets same data, fewer chars). No CCR, no marker emission.
  * TransformError: pub enum TransformError { InvalidInput{transform:&'static str,message:String}, Skipped{transform:&'static str,message:String}, Internal{transform:&'static str,message:String} }
      Error type for reformat (and offload) transforms. Constructors: invalid_input(transform,msg), skipped(transform,msg), internal(transform,msg). thiserror-derived Display strings noted in thresholds.

-- algorithm --
  1. 1. Entry: apply(&self, content: &str). `content` is the ORIGINAL untrimmed input string. Capture it; it is the source of truth for the never-inflate comparison and for the fallback return.
  2. 2. Trim: let trimmed = content.trim() (Rust str::trim removes leading+trailing Unicode whitespace, both ASCII and Unicode WS per char::is_whitespace). In Go: strings.TrimSpace(content) is the faithful equivalent (trims Unicode whitespace).
  3. 3. Empty guard: if trimmed.is_empty() { return Err(TransformError::skipped(NAME, "empty input")) }. This covers both truly empty AND whitespace-only inputs (whitespace-only trims to empty → Skipped). Skipped is a distinct error variant from InvalidInput.
  4. 4. Parse: serde_json::from_str(trimmed) → serde_json::Value. On parse error map to TransformError::invalid_input(NAME, e.to_string()) — i.e. the serde error's Display string becomes the message. NOTE: parses the TRIMMED string, not the raw content. serde_json::Value default number handling: integers fit i64/u64 are kept as integer, otherwise f64; serde_json does NOT use arbitrary_precision by default, so very large/high-precision numbers may be re-emitted in a normalized form (e.g. trailing-zero / exponent normalization). See edgeCases for the Go precision caveat.
  5. 5. Re-emit compact: serde_json::to_string(&value) → minified String (no insignificant whitespace: no spaces after ':' or ',', no newlines/indentation). On serialization error map to TransformError::internal(NAME, e.to_string()). Object key order is PRESERVED in serde_json's default Value because Map is backed by a BTreeMap by default... CORRECTION: serde_json::Map by default is a BTreeMap (alphabetically SORTED keys) UNLESS the `preserve_order` feature is enabled (which swaps to IndexMap, insertion order). Key order behavior therefore depends on the crate's feature flags — see edgeCases/notes for the decision the Go port must make.
  6. 6. Never-inflate guard: if minified.len() >= content.len() { return Ok(ReformatOutput::from_lengths(content.len(), content.to_string())) }. CRITICAL DETAIL: the comparison is minified.len() vs content.len() — i.e. the RAW (untrimmed) content length, NOT trimmed length. And the returned fallback output is content.to_string() (the RAW original, including any surrounding whitespace), with input_len=content.len(). Because output==content here, from_lengths computes bytes_saved = content.len().saturating_sub(content.len()) = 0. So when minification doesn't help, output is the verbatim original and bytes_saved == 0.
  7. 7. Success path: return Ok(ReformatOutput::from_lengths(content.len(), minified)). input_len = content.len() (RAW length), output = minified. bytes_saved = content.len().saturating_sub(minified.len()). Since we only reach here when minified.len() < content.len(), bytes_saved is strictly > 0 in this branch.
  8. 8. byte length semantics: all .len() calls are BYTE lengths (Rust String/str .len() == UTF-8 byte count, not rune/char count). The Go port MUST use len([]byte) / len(string) (Go string len is already byte count), NOT utf8.RuneCountInString. bytes_saved is a byte delta.

-- patterns --
  * NAME constant = json_minifier
      (Transform name() and the transform field in all TransformError variants.)
  * empty-skip message = empty input
      (Exact message string for the Skipped error when trimmed input is empty/whitespace-only.)
  * applies_to content type = ContentType::JsonArray
      (The single content type this reformat applies to (umbrella tag for arrays+objects).)
  * mod.rs exports = pub mod json_minifier; pub mod log_template; pub use json_minifier::JsonMinifier; pub use log_template::LogTemplate;
      (The reformats module declares two submodules (json_minifier, log_template) and re-exports JsonMinifier and LogTemplate. Confirms reformats sibling = log_template (LogTemplate) for the broader port.)
  * compact emit (serde semantics) = serde_json::to_string — separators are ',' and ':' with NO surrounding spaces; no newlines/indent
      (Defines the exact minified output shape: {"a":1,"b":2} and [1,2,3]. Go must emit with no space after ':' or ',' (default json.Marshal already does this), and SetEscapeHTML(false).)

-- thresholds --
  * NAME = "json_minifier"  :: const NAME: &str = "json_minifier". The transform's name(), and the `transform` field stamped into every TransformError (Skipped/InvalidInput/Internal).
  * never-inflate gate = minified.len() >= content.len()  :: If the compact re-emit is the same size OR LARGER than the raw original, discard the minified output and return the raw original unchanged. Uses >= (greater-OR-equal): equal-length output is also rejected in favor of the original. Compared against RAW content.len(), not trimmed length.
  * empty-skip condition = content.trim().is_empty()  :: Empty or whitespace-only input → Err Skipped("empty input"). Checked BEFORE parsing.
  * bytes_saved formula = input_len.saturating_sub(output.len())  :: Saturating subtraction (never underflows below 0). input_len is always passed as content.len() (raw). On success = raw_len - minified_len (>0). On never-inflate fallback = raw_len - raw_len = 0.
  * applies_to slice = &[ContentType::JsonArray]  :: Single-element content-type slice. JsonArray is the umbrella detector tag covering BOTH JSON arrays and JSON objects. No JsonObject entry is registered separately here.
  * Skipped Display fmt = "{transform} skipped: {message}"  :: thiserror Display for TransformError::Skipped. e.g. "json_minifier skipped: empty input".
  * InvalidInput Display fmt = "invalid input for {transform}: {message}"  :: thiserror Display for TransformError::InvalidInput. message = serde_json parse error's Display.
  * Internal Display fmt = "{transform} internal error: {message}"  :: thiserror Display for TransformError::Internal. message = serde_json serialization error's Display (effectively unreachable for valid in-memory Values).

-- edgeCases --
  - Empty AND whitespace-only inputs both produce Skipped ("empty input"), NOT InvalidInput — they trim to empty before the parse step. Tests: empty_input_skipped, whitespace_only_skipped ("   \n\t  ").
  - Invalid JSON ({not: valid) → InvalidInput with transform=="json_minifier". The message carries serde's parse error text. Test: invalid_json_errors_with_invalid_input.
  - Already-compact JSON yields bytes_saved == 0 and output == the (untrimmed) original via the never-inflate >= branch. Test: already_compact_yields_zero_savings — input {"a":1,"b":2} returns itself, 0 saved.
  - Never-inflate is a hard contract: scalar/degenerate inputs {} [] null 42 "string" {"k":"value with spaces"} must all yield output.len() <= input.len(). Test: minifier_never_grows_output. The >= guard guarantees output is never longer than the raw input.
  - The never-inflate comparison uses RAW content.len(), but the parse uses TRIMMED content. Subtle: if input has surrounding whitespace, minified (no WS) is compared to raw (with WS) — so whitespace-padded input almost always 'wins' minification. The returned fallback (when minified doesn't win) is the RAW original WITH its surrounding whitespace, not the trimmed form.
  - All lengths are UTF-8 BYTE counts. Go port: use byte length (len(s)), never rune count. bytes_saved is a byte delta.
  - Unicode is preserved through the round-trip. serde_json::to_string does NOT \u-escape non-ASCII by default — it emits raw UTF-8 (é, 🌍, ö stay as multibyte UTF-8). Go's encoding/json escapes <, >, & to < etc. AND has SetEscapeHTML(true) by default — the Go port MUST call enc.SetEscapeHTML(false) to match serde and avoid spurious inflation of HTML-ish strings. Test: unicode_round_trips.
  - Key order: serde_json::Map default backing is BTreeMap (SORTED keys) UNLESS preserve_order feature → IndexMap (insertion order). Tests only assert structural/semantic equivalence (re-parse and Value-compare) for nested/object cases, EXCEPT pretty_object_minifies which asserts exact output {"a":1,"b":2} — but a/b are already alphabetical so that test cannot distinguish sorted vs insertion order. Faithful behavior is therefore ambiguous from this file alone; see notes for the recommended Go choice.
  - No precision/big-number test exists. serde_json default (no arbitrary_precision) parses numbers into i64/u64/f64; floats may re-emit in normalized form (1.0 may stay 1.0; large floats may gain exponent form). Go's encoding/json with json.Decoder.UseNumber() preserves the original numeric literal text exactly — using UseNumber is the SAFER choice for the Go port to avoid precision/format drift, even though upstream serde does not preserve literal text by default.
  - Internal error variant is effectively unreachable: serializing an already-parsed in-memory serde_json::Value cannot normally fail. Port may treat it as a defensive/never-hit branch.
  - Whitespace definition: str::trim removes Unicode whitespace (char::is_whitespace), which includes NBSP, etc. strings.TrimSpace in Go also trims Unicode whitespace — faithful match. (ASCII-only trim would diverge on exotic inputs.)

-- notes --
  FETCH: all three sources fetched OK (json_minifier.rs verbatim, mod.rs, and the relevant ReformatOutput/ReformatTransform/TransformError defs from traits.rs). Confidence high on the minifier algorithm.

KEY-ORDER DECISION (the one real porting ambiguity): upstream's effective key order depends on whether serde_json's `preserve_order` feature is compiled in. Default serde_json sorts object keys alphabetically (BTreeMap); with preserve_order it keeps insertion order. The minifier's own tests don't disambiguate (the only exact-output object test uses already-alphabetical keys a,b). For the Go MVP: use json.Decoder with UseNumber() into a generic interface{}; but note Go's map[string]interface{} does NOT preserve insertion order and Go's json.Marshal SORTS map keys alphabetically — which happens to match DEFAULT serde_json (BTreeMap). So a straightforward Go round-trip (Unmarshal into interface{} with UseNumber, then Marshal) yields ALPHABETICALLY SORTED keys, matching upstream's DEFAULT build. Recommend documenting this and matching default serde (sorted keys). If upstream is later found to ship preserve_order, the Go port would need an ordered-map JSON re-emitter (e.g. a custom tokenizer or a third-party ordered map) — but that violates zero-dep, so MVP = sorted keys.

NUMBER PRECISION: Go's json.Decoder.UseNumber() preserves the exact numeric literal text (json.Number is the raw string), which is actually SAFER than default serde_json (which normalizes via f64/i64). Recommend UseNumber() to avoid precision drift; it will not break any upstream test and improves losslessness.

HTML ESCAPING: MUST disable Go's default HTML escaping (SetEscapeHTML(false) on a json.Encoder, since json.Marshal always escapes <>&). Without this, strings containing < > & would inflate (\\u003c etc.) — diverging from serde and potentially defeating the never-inflate guard / changing bytes_saved.

NEVER-INFLATE FIDELITY: replicate exactly — compare minified byte len vs RAW content byte len with >= , and on loss return the RAW original string (untrimmed) with bytes_saved=0. Do not return the trimmed form on the fallback path; upstream returns content.to_string() (raw).

ERROR MODEL: port TransformError as a Go error with three constructors/kinds (InvalidInput, Skipped, Internal), each carrying transform name + message, matching the Display formats. Skipped vs InvalidInput is a meaningful distinction the pipeline likely branches on (skip = not applicable, invalid = malformed) — preserve it.

NO CCR / NO BLOAT: this subsystem has zero CCR machinery, no marker, no EstimateBloat, no Confidence. Those concepts belong to OFFLOAD transforms, not reformats. The mod.rs doc-comment explicitly states 'No CCR involvement; no marker emission.'

EDGE: trim happens before empty-check and before parse, but the never-inflate comparison and fallback use RAW (untrimmed) content. Get this asymmetry right — it's the most likely porting bug.

SUBSYSTEM: reformats/log_template   ->   internal/reformats   fetchedOk= True
====================================================================================================
appliesTo: ['build']

estimateBloat: n/a (reformat, not offload — no EstimateBloat()/structural-sniff). NOTE: there IS a separate `bloat.log` LogBloatConfig in config.rs used by a DIFFERENT log-domain bloat estimator at the orchestrator level (min_lines=50), but log_template.rs itself contains no bloat estimator. Gating here is purely the up-front guards: empty-input skip, lines.len() < min_lines (20) skip, and the post-hoc never-inflate length check.
confidence: n/a (reformat, not offload — no Confidence() function)
ccrMarker: n/a — this is a Reformat (lossless), not an Offload. No CCR marker emitted, no cache_key, no store handle. Output bytes are semantically equivalent to input (original fully reconstructible from template + variant table). min_compression_ratio_for_ccr gate does not exist here. The only "ratio gate" is the never-inflate guard: if output.len() >= input.len(), return original verbatim.

-- publicApi --
  * LogTemplate: pub struct LogTemplate { config: LogTemplateConfig }
      The reformat transform. Holds config knobs. Constructed via LogTemplate::new(config).
  * LogTemplate::new: pub fn new(config: LogTemplateConfig) -> Self
      Constructor storing the config.
  * name: fn name(&self) -> &'static str  // returns "log_template"
      Transform identity string (const NAME = "log_template").
  * applies_to: fn applies_to(&self) -> &[ContentType]  // &[ContentType::BuildOutput]
      Declares this transform applies only to BuildOutput content type.
  * apply: fn apply(&self, content: &str) -> Result<ReformatOutput, TransformError>
      Main entry. Tokenizes lines, mines consecutive same-template runs, emits template-header + variant-table blocks (or verbatim), returns ReformatOutput{output, bytes_saved} or TransformError::Skipped.
  * LogTemplateConfig: pub struct LogTemplateConfig { pub min_lines: usize, pub min_run: usize, pub similarity_threshold: f32, pub min_constant_tokens: usize }
      Four tuning knobs. Defaults (from config/pipeline.toml [reformat.log_template]): min_lines=20, min_run=3, similarity_threshold=0.4, min_constant_tokens=2.
  * ReformatOutput::from_lengths: pub fn from_lengths(input_len: usize, output: String) -> Self  // bytes_saved = input_len.saturating_sub(output.len())
      Result wrapper. bytes_saved uses saturating_sub so it is 0 (never negative) when output >= input.
  * TransformError::skipped: pub fn skipped(transform: &'static str, message: impl Into<String>) -> Self
      Constructs Skipped variant; used for empty-input and below-min_lines early returns.

-- algorithm --
  1. GUARD 1 (empty): if content.is_empty() return Err(TransformError::skipped("log_template", "empty input")).
  2. SPLIT LINES: lines = content.lines() collected into Vec<&str>. NOTE: str::lines() splits on '\n', strips a trailing '\n' (and a preceding '\r' for \r\n), and does NOT yield a trailing empty element if content ends with '\n'. Track separately whether content.ends_with('\n') for the trailing-newline fixup at the end.
  3. GUARD 2 (min_lines): if lines.len() < config.min_lines (default 20) return Err(TransformError::skipped("log_template", "input below min_lines")). Comparison is on LINE COUNT, not byte length.
  4. TOKENIZE ALL: tokenized = lines.map(tokenize) where tokenize(line) = line.split_whitespace().collect::<Vec<&str>>(). split_whitespace semantics: splits on runs of Unicode whitespace, trims leading/trailing whitespace, collapses consecutive whitespace, yields NO empty tokens. A blank or whitespace-only line yields an EMPTY token vec. UTF-8 multibyte non-whitespace chars pass through intact as part of tokens.
  5. INIT STATE: output = String with capacity content.len(); next_template_id = 1usize; run: Option<Run> = None. (Run = { indices: Vec<usize>, template: Vec<Option<String>> } where Some(tok)=constant-so-far at that position, None=wildcard/varies.)
  6. MAIN LOOP over (i, tokens) in tokenized.iter().enumerate():
  7.   CASE A — tokens.is_empty() (blank/whitespace-only line): if a run is active, take it and flush_run(...); then push the ORIGINAL line text lines[i] + '\n' to output; continue. (Blank lines ALWAYS break runs and are emitted verbatim — never collapsed.)
  8.   CASE B — run is active AND extends_run(run, tokens, similarity_threshold) is true: push i into run.indices and merge_into_template(run.template, tokens). (Extend the current run.)
  9.   CASE C — otherwise (no active run, OR active run that this line does NOT extend): if a run is active, take+flush it; then start a fresh run = Run::start(i, tokens) (indices=[i], template = each token wrapped Some(token.to_string())).
  10. END-OF-INPUT FLUSH: after the loop, if a run is still active, take+flush it.
  11. TRAILING-NEWLINE FIXUP: if content.ends_with('\n') do nothing (output already ends in '\n' from the last push). Else if output.ends_with('\n') pop() the final '\n' (so output does not grow a newline the input lacked). NOTE: every emit path (verbatim line, blank line, template header, variant row) appends a '\n', so output normally ends with '\n'.
  12. NEVER-INFLATE GUARD: if output.len() >= content.len() (byte length, >= is inclusive) return Ok(ReformatOutput::from_lengths(content.len(), content.to_string())) — i.e. fall back to the ORIGINAL content unchanged. Otherwise return Ok(ReformatOutput::from_lengths(content.len(), output)).
  13. extends_run(run, tokens, sim_threshold) -> bool: (1) if tokens.len() != run.template.len() return false (token-COUNT must match exactly). (2) len = tokens.len() as f32. (3) matches=0; for each (pos, tok): if run.template[pos] is Some(constant) and constant==tok -> matches+=1; if run.template[pos] is None (already wildcard) -> matches+=1 (wildcard counts as a match); if Some(constant) and constant!=tok -> no increment. (4) return (matches as f32 / len) >= sim_threshold. Uses f32 arithmetic and >= (inclusive).
  14. merge_into_template(template, tokens): for each (pos, tok): if template[pos] is Some(constant) and constant != tok, set template[pos] = None (demote to wildcard). Positions already None stay None. Lengths are guaranteed equal because extends_run gated on equal token count before this is called.
  15. flush_run(run, lines, tokenized, cfg, next_template_id, out): compute constant_count = count of template slots that are Some; varying_count = template.len() - constant_count. collapse = (run.indices.len() >= cfg.min_run) AND (constant_count >= cfg.min_constant_tokens) AND (varying_count > 0). ALL THREE must hold.
  16.   IF NOT collapse: emit each line verbatim — for &i in run.indices: push lines[i] + '\n'. Return.
  17.   IF collapse: emit header: "[Template T" + template_id + ": " + then for each (pos, slot) in template (space-separated, space pushed before every pos>0): Some(constant)->push constant, None->push WILDCARD (the literal string "<*>"); then "] (" + run.indices.len() + " occurrences)\n". Increment next_template_id AFTER capturing template_id (template_id = *next_template_id; *next_template_id += 1). Template IDs are global, assigned in flush order (input order), starting at 1.
  18.   THEN emit variant table: for &i in run.indices: for each (pos, slot) where slot.is_none() (i.e. wildcard positions only, in ascending position order), push tokenized[i][pos], space-separated (space before all but the first emitted variant token); end the row with '\n'. One row per original line; constant positions are NOT emitted in rows (they live in the header). Reconstruction = splice the row's variant tokens back into the header template at the wildcard positions in order.

-- patterns --
  * WILDCARD = <*>
      (Literal sentinel string emitted in the template header for each varying (None) position. const WILDCARD: &str = "<*>". Variant rows do NOT contain this; they contain the actual per-line token for that position.)
  * NAME = log_template
      (const NAME: &str = "log_template". Transform name + the `transform` field in Skipped errors.)
  * tokenizer = line.split_whitespace()
      (The ONLY 'pattern' — there are NO regexes in this transform. Tokenization is Rust str::split_whitespace (split on runs of Unicode whitespace, trim ends, collapse runs, no empty tokens). The module doc's mention of masking timestamps/numbers/hex/UUIDs/IPs/paths is NOT done by regex — variability is discovered purely positionally by merge_into_template demoting positions that differ across the run to wildcards. There are NO masking regexes for timestamps/numbers/hex/UUID/IP/path; that masking is emergent from positional token comparison, not pattern matching.)
  * header_format = [Template T<id>: <tok-or-<*> space-joined>] (<N> occurrences)
      (Exact template-header output format. Example: '[Template T1: <*> INFO worker-<*> processing job <*>] (50 occurrences)'. Brackets, the literal 'Template T', integer id, ': ', space-joined template tokens, '] (', integer occurrence count, ' occurrences)', then newline.)
  * variant_row_format = <varying-token> [SPACE <varying-token> ...]\n
      (One row per original line: only the wildcard-position tokens, space-separated, in ascending position order, terminated by '\n'. Empty wildcard set is impossible here because collapse requires varying_count>0.)

-- thresholds --
  * min_lines = 20  :: Default LogTemplateConfig.min_lines. If lines.len() < min_lines, skip the transform entirely (TransformError::Skipped "input below min_lines"). Counted in LINES, not bytes. (Tested: a 3-line input skips.)
  * min_run = 3  :: Default LogTemplateConfig.min_run. A run only collapses if run.indices.len() >= min_run. Runs shorter than 3 lines are emitted verbatim. Floor below which template-header overhead exceeds savings.
  * similarity_threshold = 0.4  :: Default LogTemplateConfig.similarity_threshold (f32). In extends_run, (matches / token_count) >= 0.4 (inclusive) required for a same-token-count line to extend the run. Drain's published default; wildcard positions count as matches. 40% positional match.
  * min_constant_tokens = 2  :: Default LogTemplateConfig.min_constant_tokens. A run only collapses if its template has constant_count >= 2 (positions that stayed constant, i.e. still Some). An all-wildcard or single-anchor template carries no signal → emit verbatim.
  * varying_count_gt_0 = >0  :: Third collapse condition (not configurable): the template must have at least one wildcard (varying_count = template.len() - constant_count > 0). If every position is constant there is nothing to dedup via a variant table, so emit verbatim. Also guarantees variant rows are non-empty.
  * never_inflate_guard = output.len() >= content.len()  :: Post-processing byte-length check (inclusive >=). If the mined output is not strictly smaller than input, discard it and return the ORIGINAL content unchanged. Guarantees the transform never grows output.
  * first_template_id = 1  :: next_template_id starts at 1usize; template IDs (T1, T2, ...) assigned globally in flush/input order, incrementing by 1 each collapsed block. Non-collapsed runs do NOT consume an ID.
  * output_capacity = content.len()  :: output String pre-allocated with capacity == input byte length (perf only, no behavioral effect).

-- edgeCases --
  - Empty input ('') → Err Skipped "empty input" (checked before line splitting).
  - lines.len() < min_lines (20) → Err Skipped "input below min_lines". Counted in lines.
  - Blank or whitespace-only lines (tokenize → empty vec) ALWAYS break the active run (flush it first) and are emitted verbatim (original lines[i] text). Runs NEVER bridge a blank line (tested: blank_lines_break_runs).
  - Order preservation: only CONSECUTIVE same-template lines collapse; runs are flushed in input order, so template blocks and verbatim lines appear in original temporal order. Template IDs increase monotonically with input position.
  - Token-count mismatch immediately disqualifies a line from extending a run (extends_run returns false before similarity math) — different-length lines never merge.
  - Wildcard positions count as matches in the similarity calc (None → matches+=1), so once a position varies it can never cause a later same-shaped line to fail similarity on that position; this makes runs 'sticky'.
  - Three-part collapse gate ALL required: len>=min_run AND constant_count>=min_constant_tokens AND varying_count>0. Failing any one → verbatim emit of that run.
  - All-varying template (every position differs across the run, e.g. '0 1 2' / '1 2 3' ...) → constant_count=0 < 2 → emitted verbatim, no [Template ...] header (tested: template_with_no_constants_emits_verbatim).
  - Lossless guarantee: original line = template with each wildcard replaced, in order, by the row's space-joined variant tokens. Constant tokens live only in the header; variant rows carry only varying-position tokens. Round-trip reconstructs original exactly (tested).
  - Never-inflate: if mined output byte length >= input byte length, return ORIGINAL content (not the mined output). Output byte length is therefore always <= input byte length; bytes_saved (saturating_sub) is >= 0.
  - Trailing newline handling: str::lines drops a trailing '\n'. If input ended with '\n', output already ends with one (every emit appends '\n') — left as-is. If input did NOT end with '\n', the final '\n' is popped so output doesn't gain a newline. (\r\n: lines() strips the \r too; CR handling is whatever str::lines does — the implementer should mirror split-on-\n with optional trailing-\r strip.)
  - UTF-8: tokenizer is split_whitespace (Unicode-whitespace-aware). Multibyte non-whitespace chars (emoji 🔥, accented héllo/wörld) survive verbatim inside tokens, including when they are constant template anchors (tested: unicode_tokens_survive). Length comparisons use BYTE length (str::len), token comparisons use string equality.
  - NO regex, NO heap-scanning estimator, NO byte-prefix sniff: a single O(n × tokens_per_line) forward pass. Variability detection is purely positional token equality, not pattern-based masking despite the module doc's timestamp/UUID/IP/path framing.
  - next_template_id is incremented only inside the collapse branch of flush_run, so a verbatim (non-collapsed) run does not consume a Tn id; ids stay contiguous across collapsed blocks only.

-- algorithm --
  1. STAGE 0 (setup): LogCompressor holds config (Default = documented constants) and three precompiled regexes used for normalization: digit_re = \d+ , hex_re = 0x[0-9a-fA-F]+ , path_re = /[\w/]+/ . compress(content) splits content into lines (by \n).
  2. STAGE 1 Format detection (FormatDetector): scan the FIRST 100 lines (sample cap). Count substring marker hits per format and pick the format with the highest count; ties / no hits fall through to Generic. Markers: Pytest = ['=== FAILURES','=== ERRORS','=== test session','=== short test summary','PASSED [','FAILED [','ERROR [','SKIPPED [','collected ']; Npm = ['npm ERR!','npm WARN','npm info','npm http']; Cargo = ['Compiling ','Finished ','Running ','warning: ','error[E']; Jest = ['PASS ','FAIL ','Test Suites:']; Make = ['make[','make:','gcc ','g++ ','clang ']; else Generic. (Exact tie-break ordering is summarizer-paraphrased; treat highest-count-wins, Generic fallback as authoritative.)
  3. STAGE 2 Per-line classification (parse_lines): for each line build a LogLine. (a) LEVEL detection by keyword substring/word-boundary match, checked in precedence order Error > Fail > Warn > Info > Debug > Trace, else Unknown. Keyword sets (case-sensitive variants, matched with word boundaries so they are not part of a larger alnum/underscore token): Error = ERROR,error,Error,FATAL,fatal,Fatal,CRITICAL,critical ; Fail = FAIL,FAILED,fail,failed,Fail,Failed ; Warn = WARN,WARNING,warn,warning,Warn,Warning ; Info = INFO,info,Info ; Debug = DEBUG,debug,Debug ; Trace = TRACE,trace,Trace. (Upstream uses an Aho-Corasick automaton + word-boundary post-filter; a Go port may use a simple per-keyword contains-with-boundary check in the same precedence order.) (b) is_stack_trace: flavor-aware membership — Python: line is part of a 'Traceback (most recent call last)' block, or matches a frame line containing 'File "' AND '", line ' ending in a digit (indented continuation lines remain in-frame across blanks); JavaScript: trimmed line starts_with 'at ', contains '(' and ')', and has a :<digits>:<digits> suffix; Java: starts_with 'at ', contains '(', package body chars limited to alnum/./_/$ ; Rust: starts_with '--> ' with :<digits>:<digits> suffix; Go: starts with digits then ':' then '0x' + hex digits. (c) is_summary: line matches a summary heuristic — starts with '===' or '---'; or starts with <digits><space> then one of passed/failed/skipped/error/warning; or starts with Test/Tests/Suite/Suites (optional ':') followed by digits; or starts with TOTAL/Total/Summary; or starts with Build/Compile/Test and contains succeeded/failed/complete. (Exact summary booleans are summarizer-paraphrased — treat as the intended heuristic, verify against fixtures.)
  4. STAGE 3 Per-line scoring (score_log_line): level_score = {Error|Fail => 1.0, Warn => 0.5, Info|Unknown => 0.1, Debug => 0.05, Trace => 0.02}; stack_boost = is_stack_trace ? 0.3 : 0.0 ; summary_boost = is_summary ? 0.4 : 0.0 ; score = (level_score + stack_boost + summary_boost).min(1.0_f32). Store on the LogLine.
  5. STAGE 4 Adaptive budget: compute an adaptive total-lines target via crate::transforms::adaptive_sizer::compute_optimal_k (separate subsystem). This yields the effective cap, bounded by config.max_total_lines = 100. (The adaptive_sizer formula is out of scope for this file; port log_compressor to call into it and clamp by max_total_lines.)
  6. STAGE 5 Category selection + context windows + final cap: group classified lines into categories and select using config knobs, collecting selected line indices into an ordered/dedup set. ERRORS: keep up to max_errors=10, honoring keep_first_error/keep_last_error (first + last + top-scoring), each with a context window of error_context_lines=3 surrounding lines. FAILS: selected similarly. WARNINGS: keep up to max_warnings=5, deduped when dedupe_warnings=true (see dedup below). STACK TRACES: keep up to max_stack_traces=3 blocks, each truncated to stack_trace_max_lines=20 lines. SUMMARIES: kept when keep_summary_lines=true. After gathering, if total selected exceeds the adaptive cap / max_total_lines, sort the selected set by score descending and truncate to the cap. Preserve original line order in final output (selection by index, then re-sorted to source order for emission).
  7. DEDUP (warnings) detail: key = normalize_for_dedupe(content). normalize splits content at the first ':' or '=' into prefix + suffix; prefix is preserved verbatim; suffix is normalized by applying the 3 regexes in strict sequence: digit_re -> 'N', then hex_re -> 'ADDR', then path_re -> '/PATH/'. The resulting prefix+normalized-suffix string is the dedup key, stored in a BTreeSet<String>; duplicates (same key) are dropped. NOTE: there is NO 'repeated N times' / '(N more)' literal emitted by this engine — dedup silently removes duplicate lines (this corrects an assumption in the task brief; the brief's expected 'repeated N times' literal does NOT exist in upstream).
  8. STAGE 6a Output formatting (format_output): join selected lines (in source order) with '\n'. When lines were omitted, append a footer summarizing omitted counts by level. Footer literal (summarizer-paraphrased): '[{} lines omitted: {}]' where the second field is a comma-separated list of nonzero level counts among ERROR/FAIL/WARN/INFO, e.g. '[3 lines omitted: 1 ERROR, 1 WARN, 2 INFO]'. (Treat exact footer wording as needing fixture verification.)
  9. STAGE 6b Optional CCR storage/marker: compute compression_ratio = compressed_line_count / original_line_count (float). Retrieve more: hash={}]' with args (original_line_count, selected.len(), key); set result.ccr_key = Some(key). Otherwise ccr_key = None and no marker appended. Return LogCompressionResult.

  11. STAGE 8 (CCR gate + marker): if enable_ccr AND (compressed_line_count as f64) < (original_line_count as f64) * min_compression_ratio_for_ccr (0.8): key = md5_hex_24(content) [MD5 of UTF-8 bytes, lowercase hex, truncated to first 24 chars]. Append '\n' then the literal marker line: "[{original_line_count} lines compressed to {compressed_line_count}. Retrieve full diff: hash={key}]". If store is Some, store.put(key, content). cache_key = Some(key); stats.cache_key_emitted=true. ELSE if !enable_ccr: ccr_skipped_reason="ccr disabled". ELSE: ccr_skipped_reason = format "compression ratio {ratio:.3} above threshold {threshold:.3}" where ratio = compressed_line_count/original_line_count (or 1.0 if original==0).
  12. STAGE 9 (final stats + return): output_lines = compressed_line_count (the PRE-marker count); compression_ratio = compressed_line_count/original_line_count (1.0 if original==0); processing_duration_us set. Build DiffCompressionResult with compressed=compressed_output (which now MAY include the marker line, so the actual string has compressed_line_count+1 lines when marker emitted — intentional, matches Python). Return (result, stats).
  13. PARSING (parse_diff): iterate lines in order, tracking current_file: Option<DiffFile> and current_hunk: Option<DiffHunk>. (1) If is_diff_header(line) [matches diff --git OR diff --combined OR diff --cc]: flush current_hunk into current_file, flush current_file into files, start new DiffFile{header=line, old_file="", new_file="", empty hunks, all bools false, empty rename_lines, None mode/binary lines}; continue. (2) Else if current_file is None: push line to pre_diff_lines (verbatim); continue. (3) File-level markers (checked while current_file is Some, NOT continue-guarded as a block — falls through to the regex checks below): if line.starts_with("new file mode") → is_new_file=true, original_new_file_mode_line=Some(line); else if starts_with("deleted file mode") → is_deleted_file=true, original_deleted_file_mode_line=Some(line); else if starts_with any of "rename "/"similarity "/"copy "/"dissimilarity " → is_renamed=true, push line to rename_lines; else if binary_regex matches → is_binary=true, original_binary_line=Some(line). (4) If old_file_regex matches ("--- a/... " or "--- /dev/null") → set f.old_file = line (FULL line including "--- " prefix); continue. (5) If new_file_regex matches ("+++ b/..." or "+++ /dev/null") → set f.new_file = full line; continue. (6) If hunk_header_regex matches → flush current_hunk into file, start new DiffHunk{header=line, empty lines, additions=0, deletions=0, context_lines=0, score=0.0}; continue. (7) Else if current_hunk is Some (hunk content): if line.starts_with('+') AND NOT starts_with("+++") → additions+=1, push line; else if starts_with('-') AND NOT starts_with("---") → deletions+=1, push line; else if starts_with(' ') OR line.is_empty() → context_lines+=1, push line; else ("other" line e.g. backslash no-newline marker) → push line verbatim (no counter bump). At end: flush trailing current_hunk and current_file. parse_warnings stays empty in this version (declared but never appended to).
  14. SCORING (score_hunks): context_lower = context.to_lowercase(); context_words = context_lower.split_whitespace() (whitespace-split). For each hunk: score=0.0. score += (additions + deletions) * 0.03 (SCORE_CHANGE_DENSITY_WEIGHT); if score > 0.3 (CAP) → score = 0.3. hunk_content_lower = hunk.lines.join("\n").to_lowercase(). For each word in context_words: if word.len() > 2 (SCORE_CONTEXT_MIN_WORD_LEN) AND hunk_content_lower.contains(word) → score += 0.2 (each matching word, cumulative). For each of the 3 priority_patterns (case-insensitive regex): if pat.is_match(hunk_content_lower) → score += 0.3 and BREAK (only ONE priority boost per hunk). Finally if score > 1.0 (SCORE_TOTAL_CAP) → score = 1.0. Store hunk.score.
  15. HUNK SELECTION (select_hunks(hunks, max_per_file)): if hunks.len() <= max_per_file → return (hunks, []) unchanged. if hunks empty → ([],[]). Otherwise: index every hunk with its original position (enumerate). first = index 0 (always kept). last = pop final element if any remain (kept). middle = everything between. remaining_slots = if last is Some: max_per_file - 2, else max_per_file - 1 (saturating_sub). Sort middle by hunk.score DESCENDING (partial_cmp, NaN→Equal). Partition middle by rank: kept_middle = rank < remaining_slots, dropped_middle = rest. Reassemble selected = [first] + kept_middle + (last if any). Then SORT selected by extract_line_number(header) ASCENDING to restore appearance order. Return (selected hunks, dropped_middle hunks). NOTE: first and last hunks are ALWAYS retained; only middle hunks compete on score.
  16. extract_line_number(header): apply hunk_new_range_regex (matches first \+(\d+) anywhere) → parse capture group 1 as usize; on any failure return 0. Works for @@ and @@@ headers (the '+N' new-file start line).
  17. CONTEXT TRIM (reduce_context(hunk, max_context)): change_positions = indices of lines where line.starts_with('+') OR starts_with('-'). If change_positions empty: keep first min(max_context, lines.len()) lines, set additions=0, deletions=0, context_lines=take; return. Else build keep:BTreeSet<usize>: for each change pos: insert pos; insert lo..pos where lo = pos.saturating_sub(max_context); insert (pos+1)..hi where hi = min(pos+max_context+1, lines.len()). THEN: for every line index i where line.starts_with('\\') (backslash) → keep.insert(i) UNCONDITIONALLY (preserves '\ No newline at end of file' regardless of distance). Build new_lines from sorted keep set (BTreeSet iterates ascending), recount: starts_with('+')→additions++, starts_with('-')→deletions++, else context_lines++. Return new hunk preserving header and score.
  18. OUTPUT FORMAT (format_output): out_lines = []. (1) Emit every pre_diff_line verbatim. (2) For each file: push f.header; push each rename_line verbatim; if is_new_file push literal "new file mode 100644" (HARDCODED — loses original mode); else if is_deleted_file push "deleted file mode 100644"; if is_binary push literal "Binary files differ" (HARDCODED — loses filenames) and CONTINUE (skip hunks for binary files); else: if old_file non-empty push it, if new_file non-empty push it, then for each hunk push hunk.header then every hunk line. (3) Footer: if hunks_removed > 0 OR files_affected > 0: build parts = ["{files_affected} files changed", "+{total_additions} -{total_deletions} lines"] and if hunks_removed > 0 also "{hunks_removed} hunks omitted"; push "[{parts joined by ', '}]". Result = out_lines.join("\n").

-- patterns --
  * hunk_header_regex = ^(?:@@ -\d+(?:,\d+)? \+\d+(?:,\d+)? @@|@@@ -\d+(?:,\d+)? -\d+(?:,\d+)? \+\d+(?:,\d+)? @@@|@@@@ -\d+(?:,\d+)? -\d+(?:,\d+)? -\d+(?:,\d+)? \+\d+(?:,\d+)? @@@@)(.*)$
      (Matches a hunk header: regular @@ (1 -range, 1 +range), 3-way merge @@@ (2 -ranges, 1 +range), 4-way merge @@@@ (3 -ranges, 1 +range). The trailing (.*) captures the section heading. Octopus merges with 5+ @ would slip through (rare).)
  * hunk_new_range_regex = \+(\d+)
      (Extracts the new-file start line number (+N) from any hunk header for the in-order resort after middle-hunk selection.)
  * diff_git_regex = ^diff --git a/(.+) b/(.+)$
      (Matches the standard git file-section header.)
  * diff_combined_regex = ^diff --combined (.+)$
      (Merge-commit combined-diff header (single path). Bug-fix: previously unrecognized.)
  * diff_cc_regex = ^diff --cc (.+)$
      (Alternate merge-commit header form. is_diff_header = git OR combined OR cc.)
  * old_file_regex = ^--- (a/(.+)|/dev/null)$
      (Old-file line: --- a/path or --- /dev/null. Stored as full line (incl '--- ').)
  * new_file_regex = ^\+\+\+ (b/(.+)|/dev/null)$
      (New-file line: +++ b/path or +++ /dev/null. Stored as full line.)
  * binary_regex = ^Binary files .+ differ$
      (Binary marker line. Original captured; emit hardcodes 'Binary files differ'.)
  * priority_pattern_error = (?i)\b(error|exception|fail(?:ed|ure)?|fatal|critical|crash|panic)\b
      (ERROR-class priority pattern (scoring boost). Case-insensitive.)
  * priority_pattern_importance = (?i)\b(important|note|todo|fixme|hack|xxx|bug|fix)\b
      (IMPORTANCE-class priority pattern (scoring boost).)
  * priority_pattern_security = (?i)\b(security|auth|password|secret|token)\b
      (SECURITY-class priority pattern (scoring boost).)
  * file_level_marker_prefixes = starts_with: 'new file mode' | 'deleted file mode' | 'rename ' | 'similarity ' | 'copy ' | 'dissimilarity '
      (Literal prefixes detected during parse for mode/rename markers (note trailing spaces on rename/similarity/copy/dissimilarity).)
  * hunk_content_dispatch = '+' (not '+++') => addition; '-' (not '---') => deletion; ' ' or empty => context; else => other(verbatim)
      (Per-line classification inside a hunk body.)
  * no_newline_marker_keep = line.starts_with('\\')
      (Any backslash-prefixed line (e.g. '\ No newline at end of file') is ALWAYS kept by reduce_context regardless of distance from a change.)
  * ccr_marker_format = [{original_line_count} lines compressed to {compressed_line_count}. Retrieve full diff: hash={md5_hex_24}]
      (Exact CCR retrieval marker emitted (appended after '\n') when ratio gate passes.)
  * summary_footer_format = [{N} files changed, +{A} -{D} lines{, {H} hunks omitted}]
      (Footer line. 'hunks omitted' clause only when hunks_removed > 0. Footer only emitted when hunks_removed>0 OR files_affected>0.)
  * label_format = {old_file} -> {new_file}
      (File label used in files_dropped, hunks_dropped_per_file keys, and lossy-emit stats. Uses the FULL '--- a/...' / '+++ b/...' lines (which may be empty strings).)

-- thresholds --
  * max_context_lines = 2  :: Context lines kept on EACH side of every +/- change line during reduce_context.
  * max_hunks_per_file = 10  :: Per-file hunk cap. When exceeded, keep first + last + top-scored middle (remaining_slots = cap-2 when last exists).
  * max_files = 20  :: Whole-diff file cap. When exceeded, sort files by (additions+deletions) desc and keep top 20; rest dropped from output.
  * min_lines_for_ccr = 50  :: Misnomer: gates the WHOLE compression path. If original_line_count < 50, input returned unchanged (no parse, no summary, no CCR).
  * min_compression_ratio_for_ccr = 0.8  :: CCR marker gate. Marker emitted only if compressed_line_count < original_line_count * 0.8 (i.e. >20% line savings). Python hardcodes 0.8.
  * enable_ccr = true  :: If false, never emit CCR marker (ccr_skipped_reason="ccr disabled").
  * always_keep_additions = true  :: RESERVED/unused. Algorithm always keeps + lines regardless.
  * always_keep_deletions = true  :: RESERVED/unused. Algorithm always keeps - lines regardless.
  * SCORE_CHANGE_DENSITY_WEIGHT = 0.03  :: Per-change-line weight: base score = (additions+deletions)*0.03.
  * SCORE_CHANGE_DENSITY_CAP = 0.3  :: Cap on the change-density base term before other boosts.
  * SCORE_CONTEXT_WORD_WEIGHT = 0.2  :: Boost per matching context-query word found in hunk content (cumulative).
  * SCORE_CONTEXT_MIN_WORD_LEN = 2  :: Context words must have len > 2 (strictly greater) to count; filters stop-words.
  * SCORE_PRIORITY_PATTERN_BOOST = 0.3  :: Boost added once if ANY priority pattern matches (break after first match).
  * SCORE_TOTAL_CAP = 1.0  :: Final clamp on total hunk score.
  * md5_truncate_len = 24  :: CCR cache_key = md5 hex digest truncated to first 24 chars (matches hashlib.md5(s.encode()).hexdigest()[:24]).

-- edgeCases --
  - Pass-through paths return content VERBATIM (compressed = original content exactly) with files_affected=0 and no cache_key. Two triggers: original_line_count < 50, OR parse produced zero files. compression_ratio reported as 1.0 in both.
  - Line counting MUST use split('\n').count() (Python str.split semantics): empty string => 1 line; 'a\n' => 2 lines; trailing newline produces a trailing empty element. NEVER use lines()/splitlines() which differ on trailing newline. This is load-bearing for byte-parity counts.
  - compressed_line_count is captured BEFORE the CCR marker is appended. When the marker is emitted, the actual compressed string has compressed_line_count+1 newline-split lines, but the reported field and the marker's own 'compressed to N' text both use the pre-append count. Recounting after append breaks parity by 1.
  - total_additions/total_deletions in the result and footer are summed from ORIGINAL (pre-trim, pre-hunk-cap) hunk counts via file.total_additions()/total_deletions(), computed at the top of the per-file loop — NOT recounted from the trimmed/selected hunks.
  - select_hunks ALWAYS keeps first and last hunk; only middle hunks compete on score. remaining_slots = cap-2 when a distinct last exists, else cap-1 (saturating_sub guards against cap<2). After selection, hunks are re-sorted by extract_line_number(header) ascending to restore source order.
  - reduce_context on a hunk with NO +/- changes keeps only the first min(max_context, len) lines and zeroes additions/deletions. With changes, keep set is a BTreeSet (dedups + sorts ascending) so output line order is preserved and overlapping context windows merge naturally.
  - Backslash-prefixed lines ('\ No newline at end of file') are force-kept by reduce_context regardless of proximity to a change. These count as context_lines in the recount (they are not '+' or '-').
  - Hunk content dispatch guards: '+' classified as addition ONLY if NOT '+++' (the '+++ b/file' header is excluded); '-' as deletion ONLY if NOT '---'. Empty lines count as context (' ' OR is_empty). Any other non-matching line is appended verbatim as 'other' with no counter increment.
  - Binary files: when is_binary, format_output emits literal 'Binary files differ' and SKIPS all hunks/old/new file lines for that file (continue). Original detailed binary line lost (surfaced in binary_files_simplified stat).
  - File mode emit is HARDCODED to '100644': is_new_file => 'new file mode 100644', is_deleted_file => 'deleted file mode 100644'. Original mode (e.g. 100755 executable, 120000 symlink, 160000 gitlink) is lost; surfaced in file_mode_normalizations stat. is_new_file/is_deleted_file are mutually exclusive in the emit (if/else if).
  - File cap sort is by (additions+deletions) DESC; it is NOT a stable-key total order so ties keep Rust's sort_by (stable) input order. Dropped files removed via split_off(max_files).
  - old_file/new_file are stored as the FULL line including '--- '/'+++ ' prefix (or '--- /dev/null'). The label and emit use these full strings; they can be empty if the diff lacked those header lines (e.g. combined-diff with only 'index' line) — emit skips empty old_file/new_file.
  - parse_warnings vector is declared but NEVER populated in this version (always empty), despite doc comments about octopus-merge warnings. Octopus merges (5+ @) silently fall through to the content-line branch.
  - context word matching is a plain case-insensitive substring (contains), not word-boundary; words must be len > 2 (strictly). Each matching word adds 0.2 cumulatively (a 4-word query all matching adds up to 0.8 before priority boost and the 1.0 clamp).
  - md5_hex_24 hashes the ORIGINAL full content's UTF-8 bytes (not the compressed output), lowercase hex, first 24 chars. Verified vectors: md5('hello')[:24]='5d41402abc4b2a76b9719d91', md5('')[:24]='d41d8cd98f00b204e9800998'.
  - footer emitted only when hunks_removed>0 OR files_affected>0; with files_affected>0 (the normal compressed case) it always appears. 'hunks omitted' clause appended only if hunks_removed>0.

-- algorithm --
  1. ENTRY: compress(content, context, bias) just calls compress_with_store(content, context, bias, None). With store=None, CCR is never emitted (skip reason 'no store provided'). All real work is in compress_with_store.
  2. STAGE 1 PARSE (parse_search_results): Initialize empty BTreeMap<String,FileMatches> (key = file path, so output is sorted by path). Split content on '\n'. For each raw line: let line = raw.trim(); if line.is_empty() -> continue (no stat increment). Else stats.lines_scanned += 1; call parse_match_line(line).
  3. PARSE-LINE (parse_match_line) — byte-scan, NO regex: bytes = line.as_bytes(). Compute scan_start: if bytes.len() >= 3 && bytes[0].is_ascii_alphabetic() && bytes[1] == b':' && (bytes[2] == b'\\' || bytes[2] == b'/') then scan_start = 2 (Windows drive C:\ or C:/ guard — skip past the drive colon so it is not treated as the path:line separator), else scan_start = 0.
  4. PARSE-LINE loop: i = scan_start. While i < len: if bytes[i] is b':' OR b'-': (a) if i>0 && bytes[i-1] is b':' or b'-' -> this is an adjacent separator, do i+=1; continue (skip runs of separators). (b) digits_start = i+1; advance j from digits_start while bytes[j].is_ascii_digit(). (c) If j>digits_start (at least one digit) AND j<len AND bytes[j] is b':' or b'-' (closing separator): then if i==0 return None (zero-length path rejected); file = line[..i]; line_no = parse u64 from bytes[digits_start..j] (None on failure via ?); content = line[j+1..]; return Some((file, line_no, content)). Else i += 1 and keep scanning. If loop ends with no match, return None. NOTE separators are BOTH ':' and '-' (ripgrep uses '-' for context lines), and the LEFTMOST valid <sep><digits><sep> triplet wins.
  5. PARSE accumulate: On Some((file,line_no,body)) -> out.entry(file.to_string()).or_insert_with(|| FileMatches::new(file)).matches.push(SearchMatch::new(file, line_no, body)). On None -> stats.lines_unparsed += 1. NOTE: max_files cap is NOT applied here; all parsed files are retained at parse time.
  6. STAGE 2 EARLY GUARD: After parse, if parsed.is_empty() (zero matches parsed) -> return SearchCompressionResult{ compressed = content (unchanged), original = content, original_match_count=0, compressed_match_count=0, files_affected=0, compression_ratio=1.0, cache_key=None, summaries=empty } plus stats. This is the ONLY early return / no-op guard. There is NO minimum-line-count constant.
  7. STAGE 3 COUNT: original_count = sum over all FileMatches of matches.len().
  8. STAGE 4 SCORE (score_matches): context_lower = context.to_ascii_lowercase(); context_words = context_lower.split_whitespace().filter(|w| w.len() > 2). For every match in every file: score = 0.0f32; content_lower = m.content.to_ascii_lowercase(). For each context word w: if content_lower.contains(w) -> score += 0.3. If config.boost_errors: signal = importance.score(&m.content, ImportanceContext::Search); if signal.category is Some(category) -> bump = match category { Error => 0.5, Warning => 0.4, Importance => 0.3, Security | Markdown => 0.0 }; score += bump. For each kw in config.context_keywords: if content_lower.contains(&kw.to_ascii_lowercase()) -> score += 0.4. Finally m.score = score.min(1.0).
  9. STAGE 5 SELECT (select_matches): (5a) by_score = files.iter().collect() into Vec; sort by fm.total_score() DESCENDING (partial_cmp, NaN treated as Equal). (5b) If by_score.len() > config.max_files: stats.files_dropped += (len - max_files); truncate to max_files (drops lowest-scoring files).
  10. SELECT (5c) ADAPTIVE TOTAL: build all_match_strings = for each surviving (file,fm), for each match, format!("{}:{}:{}", file, m.line_number, m.content); collect &str refs; adaptive_total = compute_optimal_k(&all_refs, bias, 5, Some(config.max_total_matches)). Args: items=all match strings, bias=bias, base_k/min=5, max=Some(max_total_matches=30). compute_optimal_k lives in adaptive_sizer module (see notes) — it returns a target match count in [5, 30] adjusted by bias and corpus size.
  11. SELECT (5d) PER-FILE FILL: selected = empty BTreeMap; total_selected = 0. Iterate files in by_score order (score desc). If total_selected >= adaptive_total: stats.matches_dropped_by_global_cap += fm.matches.len(); continue (skip whole file). Else clone fm.matches into `sorted` and sort by score DESC, tie-break by line_number ASC (b.score.partial_cmp(a.score) then a.line_number.cmp(b.line_number)).
  12. SELECT (5e) remaining_cap = config.max_matches_per_file.min(adaptive_total.saturating_sub(total_selected)). seen = empty BTreeSet<(u64,u64)>. push_unique(m) closure: key = (m.line_number, hash_u64(&m.content)); if seen.insert(key) -> push clone to file_selected, return true; else return false (dedup). Order of insertion: (1) if always_keep_first: first = fm.first(); if file_selected.len() < remaining_cap -> push_unique(first). (2) if always_keep_last && fm.matches.len() > 1: last = fm.last(); if len < remaining_cap -> push_unique(last). (3) for m in &sorted (score desc): if file_selected.len() >= remaining_cap break; push_unique(m).
  13. SELECT (5f) FINALIZE FILE: file_selected.sort_by_key(|m| m.line_number) (restore ascending line order for output). dropped_here = fm.matches.len().saturating_sub(file_selected.len()); stats.matches_dropped_by_per_file_cap += dropped_here. total_selected += file_selected.len(). selected.insert(file.clone(), FileMatches{ file, matches: file_selected }).
  14. STAGE 6 FORMAT (format_output): lines = Vec; summaries = BTreeMap. For each (file, fm) in selected (BTreeMap -> sorted by path): for each m in fm.matches push format!("{}:{}:{}", m.file, m.line_number, m.content). Then if original.get(file) has orig_fm with orig_fm.matches.len() > fm.matches.len(): omitted = diff; summary = format!("[... and {} more matches in {}]", omitted, file); push summary; summaries.insert(file, summary). Return (lines.join("\n"), summaries). This is compressed_body.
  15. STAGE 7 RATIO: compressed_count = sum of selected matches.len(). ratio = compressed_body.len() as f64 / content.len().max(1) as f64 (byte-length ratio, NOT match-count ratio).
  16. STAGE 8 CCR (only path that appends a marker): compressed = compressed_body; cache_key = None. If config.enable_ccr: if original_count < config.min_matches_for_ccr -> stats.ccr_skip_reason = Some("below min_matches_for_ccr"). else if ratio >= config.min_compression_ratio_for_ccr -> stats.ccr_skip_reason = Some("compression ratio too high") (i.e. compression not good enough — 0.8 gate, STRICT >=). else if Some(store): key = md5_hex_24(content); store.put(&key, content); marker = format!("\n[{} matches compressed to {}. Retrieve more: hash={}]", original_count, compressed_count, key); compressed.push_str(&marker); cache_key = Some(key); stats.ccr_emitted = true. else (store is None) -> stats.ccr_skip_reason = Some("no store provided"). If !enable_ccr -> stats.ccr_skip_reason = Some("ccr disabled in config").
  17. STAGE 9 RETURN: SearchCompressionResult{ compressed, original = content, original_match_count = original_count, compressed_match_count = compressed_count, files_affected = scored.len(), compression_ratio = ratio, cache_key, summaries } plus stats.
  18. HASH HELPERS: hash_u64(s) = std::collections::hash_map::DefaultHasher (SipHash, used ONLY for in-run dedup key — NOT stable across processes, fine since dedup is per-call). md5_hex_24(s) = md5::Md5 digest -> lowercase hex (format!("{:02x}")) -> truncate to 24 chars (used for the CCR retrieval key / store key).

-- patterns --
  * line_separators = b':' and b'-'
      (Both colon and dash are valid separators in the path<sep><digits><sep>content scan. Dash supports ripgrep context lines (file-NN-text). Leftmost valid triplet wins.)
  * windows_drive_guard = bytes.len() >= 3 && bytes[0].is_ascii_alphabetic() && bytes[1] == b':' && (bytes[2] == b'\\' || bytes[2] == b'/')
      (Detects a Windows drive prefix (C:\ or C:/) and sets scan_start=2 so the drive colon is NOT mistaken for the path:line separator.)
  * adjacent_separator_skip = if i > 0 && (bytes[i-1] == b':' || bytes[i-1] == b'-') { i += 1; continue; }
      (Skips runs of consecutive separators so '::' or '--' don't form a bogus empty digit group.)
  * digit_run = while j < bytes.len() && bytes[j].is_ascii_digit() { j += 1 }
      (Collects the line-number digit run after the first separator; must be non-empty (j>digits_start) and followed by a second separator.)
  * zero_length_path_reject = if i == 0 { return None }
      (Rejects lines whose first separator is at index 0 (empty path).)
  * match_output_template = "{}:{}:{}" -> file:line_number:content
      (Output rendering of each surviving match (always uses ':' even if input used '-').)
  * summary_template = "[... and {} more matches in {}]" -> omitted_count, file
      (Per-file truncation summary appended after a file's matches when some were dropped.)
  * ccr_marker_template = "\n[{} matches compressed to {}. Retrieve more: hash={}]" -> original_count, compressed_count, md5_hex_24(content)
      (Lossy CCR retrieval marker appended to compressed body when the 0.8 gate passes and a store exists.)
  * dedup_key = (m.line_number: u64, hash_u64(m.content): u64)  // hash_u64 = std DefaultHasher (SipHash)
      (Per-file dedup key inside a seen BTreeSet<(u64,u64)>; identical (line,content) pairs are dropped. NOT FNV — it is std DefaultHasher.)
  * ccr_skip_reasons = "below min_matches_for_ccr" | "compression ratio too high" | "no store provided" | "ccr disabled in config"
      (Static strings recorded in stats.ccr_skip_reason explaining why no marker was emitted.)

-- thresholds --
  * max_matches_per_file = 5  :: Per-file cap (combined with adaptive remaining budget via .min) on how many matches survive from a single file.
  * always_keep_first = true  :: Always retain the first match of each file (if budget allows) regardless of score.
  * always_keep_last = true  :: Always retain the last match of each file, but only when the file has >1 match (if budget allows).
  * max_total_matches = 30  :: Hard ceiling passed as the max arg to compute_optimal_k; adaptive_total is capped at this.
  * max_files = 15  :: Files beyond this (lowest total_score) are dropped before selection; stats.files_dropped counts them.
  * context_keywords = Vec::new() (empty by default)  :: Extra keyword list; each substring hit (case-insensitive) adds +0.4 to a match score.
  * boost_errors = true  :: Enables importance-detector category boosting in score_matches.
  * enable_ccr = true  :: Master switch for emitting the CCR retrieval marker.
  * min_matches_for_ccr = 10  :: Minimum original_count required before CCR can emit; if original_count < 10 -> skip ('below min_matches_for_ccr').
  * min_compression_ratio_for_ccr = 0.8  :: CCR gate. Marker emitted only when ratio < 0.8 (byte ratio compressed/original). If ratio >= 0.8 -> skip ('compression ratio too high'). STRICT >= comparison.
  * compute_optimal_k base/min arg = 5  :: The base/minimum k passed to compute_optimal_k(all_refs, bias, 5, Some(max_total_matches)); adaptive_total floor is 5.
  * compute_optimal_k max arg = Some(max_total_matches) = Some(30)  :: Upper bound for adaptive_total.
  * context word min length = > 2 (w.len() > 2)  :: Only context words longer than 2 chars are used for +0.3 substring scoring.
  * context word score = +0.3  :: Per matching context word substring hit.
  * Error category bump = +0.5  :: Importance category Error adds 0.5 (only when boost_errors).
  * Warning category bump = +0.4  :: Importance category Warning adds 0.4.
  * Importance category bump = +0.3  :: Importance category Importance adds 0.3.
  * Security/Markdown category bump = 0.0  :: Categories Security and Markdown add nothing.
  * config keyword score = +0.4  :: Per config context_keyword substring hit.
  * score clamp = min(1.0)  :: Final per-match score capped at 1.0.
  * md5 hex truncation = 24  :: MD5 hex digest truncated to first 24 chars for the CCR key.
  * Windows drive scan_start = 2  :: When line starts with <letter>:<\ or />, byte scan starts at index 2 to skip the drive colon.
  * empty-parse ratio = 1.0  :: compression_ratio returned when parsed.is_empty() (no-op passthrough).

-- edgeCases --
  - No estimate_bloat/confidence/applies_to/can_handle/name in this file — those belong to a separate Transform-dispatch layer. Port the compressor as a plain struct with compress/compress_with_store; wire dispatch separately.
  - ONLY no-op guard is parsed.is_empty(): if zero lines parse to matches, return content unchanged with ratio=1.0 and all counts 0. No minimum line/match count to attempt compression.
  - Windows-drive guard is precise: requires len>=3, ASCII-alpha first byte, ':' second byte, and '\' OR '/' third byte. Only then scan_start=2. A bare 'C:' with no slash is NOT guarded (would scan from 0). Implement on RAW BYTES, not chars.
  - Separators are BOTH ':' and '-'. The parser accepts file-123-content (ripgrep context) and file:123:content equally. The leftmost valid <sep><digits><sep> triplet is chosen — earlier separators that aren't followed by digits+separator are skipped.
  - Adjacent-separator skip (bytes[i-1] is sep) prevents empty digit groups; without it '::5:' style lines could misparse.
  - i==0 first-separator -> None (rejects empty path). Combined with Windows guard, 'C:\foo:10:bar' parses path='C:\foo' because scan starts at index 2.
  - Line is trim()'d before parse; empty (post-trim) lines are skipped WITHOUT incrementing lines_scanned. Non-empty unparseable lines increment lines_unparsed.
  - content slice is line[j+1..] using the TRIMMED line, so trailing/leading whitespace of the original raw line is already stripped; content itself keeps internal whitespace.
  - Output always renders with ':' separator (format!("{}:{}:{}")) regardless of whether the input used '-'. This normalizes ripgrep context-line dashes to colons — a behavioral note for parity tests.
  - BTreeMap keying by file path means files (and thus output) are emitted in lexicographic path order, NOT score order — even though SELECTION iterates in score order. format_output re-sorts by path via BTreeMap. Per-file matches are emitted in ascending line_number order.
  - always_keep_last only fires when fm.matches.len() > 1 (avoids duplicating the single match already added as first). Both first/last are gated by file_selected.len() < remaining_cap and by push_unique dedup.
  - remaining_cap = max_matches_per_file.min(adaptive_total - total_selected) using saturating_sub — when global budget is exhausted, remaining_cap can be 0 and the file contributes nothing (but the file still gets inserted with an empty matches vec, which format_output renders as just a summary line if original had matches... note: with 0 selected and original>0, summary '[... and N more...]' IS emitted).
  - compression_ratio is a BYTE ratio (compressed_body.len()/content.len().max(1)), not a match-count ratio — the CCR 0.8 gate is therefore about byte savings.
  - hash_u64 uses std DefaultHasher (SipHash, randomized? No — std DefaultHasher is deterministic with fixed keys 0,0 when constructed via ::new()). It is only used within a single compress call for dedup, so cross-process stability is irrelevant. Do NOT assume FNV.
  - compute_optimal_k is an external dependency (adaptive_sizer); its exact output shapes adaptive_total. Must port/stub it: signature compute_optimal_k(items: &[&str], bias: f64, base_k: usize, max: Option<usize>) -> usize, returning a value in [base_k, max] influenced by bias and corpus characteristics.
  - The importance detector (KeywordDetector / LineImportanceDetector, ImportanceContext::Search) is also external; score_matches depends on it for error/warning boosting. Categories: Error, Warning, Importance, Security, Markdown.

appliesTo: ['build', 'diff']

estimateBloat: THREE distinct cheap structural sniffs (all O(n) single line-walk, no full compression pass):

LOG (log_offload): if content.is_empty() -> 0.0. Walk content.lines(); for the FIRST sample_size(=100) lines only (break once sampled>=sample_size): sampled++, insert line into HashSet<&str> `unique`, run detector.score(line, ImportanceContext::Log); if signal.priority <= high_priority_threshold(=0.4) low_priority++. Then total_lines = content.lines().count() (FULL count, separate pass — used only for the floor check); if total_lines < min_lines(=50) -> 0.0; if sampled==0 -> 0.0. repetition = 1.0 - (unique.len() / sampled); dilution = low_priority / sampled; score = repetition*uniqueness_weight(0.5) + dilution*priority_dilution_weight(0.5); return score.clamp(0.0,1.0).

DIFF (diff_offload): if content.is_empty() -> 0.0. Single pass over content.lines(): total_lines++ each line. If line.starts_with("@@") { in_hunk=true; continue }. If line.starts_with("diff --git") { in_hunk=false; continue }. If line.starts_with("+++") || line.starts_with("---") { continue }. If !in_hunk { continue }. Else match first byte: b'+' or b'-' -> change++; b' ' -> context++; else nothing. After loop: if total_lines < min_lines(=50) -> 0.0; denom = context+change; if denom==0 -> 0.0; ratio = context/denom; normal = normal_context_ratio(=0.6); if ratio <= normal -> 0.0; span = 1.0 - normal; if span<=0 -> 1.0 (degenerate config); return ((ratio - normal)/span).clamp(0.0,1.0).

DIFF_NOISE: if content.is_empty() -> 0.0. total_lines = content.lines().count(); if total_lines < min_lines(=30) -> 0.0. segments = parse_segments(content); if segments empty -> 0.0. For each segment: body_bytes = sum over body_lines of (line.len()+1 for the '\n'); total_bytes += body_bytes; droppable = is_lockfile(seg.new_path) || (drop_whitespace_only_hunks && seg.body_is_whitespace_only()); if droppable droppable_bytes += body_bytes. if total_bytes==0 -> 0.0; return (droppable_bytes/total_bytes).clamp(0.0,1.0). (This DOES parse segments and call body_is_whitespace_only, so it's heavier than log/diff but still single-pass, no compression, no CCR write.)
confidence: log_offload CONFIDENCE = 0.85; diff_offload CONFIDENCE = 0.85; diff_noise CONFIDENCE = 0.9. All are `const CONFIDENCE: f32` returned verbatim from `fn confidence()`.
ccrMarker: Two distinct CCR marker forms across the three transforms. (1) log_offload + diff_offload: NO marker emitted by the offload wrapper itself — they delegate entirely to the underlying LogCompressor / DiffCompressor `compress_with_store(...)`, which embeds whatever wire marker that compressor uses and stashes the original under the SAME cache_key it returns in `result.cache_key`. The wrapper just propagates that key (or returns Skipped if `cache_key` is None). So the marker form for these two is "compressor-defined / delegated" (out of scope here). (2) diff_noise: emits its OWN markers directly. Per dropped segment it inlines a CELL marker `[diff_noise: <reason> hunks dropped (<N> lines)]\n` where reason is the literal "lockfile" or "whitespace-only" and N = seg.body_lines.len(); then ONE trailing canonical CCR marker for the whole output: `\n[diff_noise CCR: hash=<key>]` where key = md5_hex_24(content). There is NO explicit min_compression_ratio_for_ccr ratio gate in diff_noise — the gate is purely structural: it Skips unless `dropped_any` is true AND `output.len() < content.len()` (it never inflates). The estimate_bloat score (droppable_bytes/total_bytes) is gated externally by the orchestrator's `bloat_threshold` (default 0.5). For log/diff offloads the "min ratio gate" lives inside the wrapped compressor (e.g. LogCompressor min_lines_for_ccr=50, DiffCompressor min_lines_for_ccr=50) and surfaces as cache_key=None → Skipped.

-- algorithm --
  1. === LogOffload.apply ===
  2. 1. Call compressor.compress_with_store(content, self.bias, Some(store)) -> (result, stats). The bias (default 0.0, or value set via WithBias) feeds the compressor's adaptive sizer; empty query in ctx maps to 0.0 in orchestrator wiring.
  3. 2. If result.cache_key is None: reason = stats.ccr_skip_reason or fallback literal "no cache_key emitted"; return Err(Skipped("log_offload", reason)). Do NOT fabricate a key.
  4. 3. Else return OffloadOutput::from_lengths(content.len(), result.compressed, key). The compressor itself stashed the original under that key.
  5. === DiffOffload.apply ===
  6. 1. Call compressor.compress_with_store(content, &ctx.query, Some(store)) -> (result, _). Note: passes ctx.query (NOT a bias) to the diff compressor.
  7. 2. If result.cache_key is None: return Err(Skipped("diff_offload", "diff compressor did not emit a cache_key")).
  8. 3. Else return OffloadOutput::from_lengths(content.len(), result.compressed, key).
  9. === DiffNoise.apply (fully self-contained, no wrapped compressor) ===
  10. 1. segments = parse_segments(content). If empty -> Err(Skipped("diff_noise", "no diff sections")).
  11. 2. output = String (capacity content.len()); dropped_any = false.
  12. 3. For each segment in order: (a) emit ALL header_lines verbatim, each followed by '\n' (header = diff --git line + everything up to but NOT including the first @@ — i.e. index/mode/---/+++ lines). (b) drop_lockfile = is_lockfile(seg.new_path); drop_whitespace = drop_whitespace_only_hunks && seg.body_is_whitespace_only(). (c) If drop_lockfile OR drop_whitespace: reason = drop_lockfile ? "lockfile" : "whitespace-only" (lockfile takes precedence); append literal "[diff_noise: " + reason + " hunks dropped (" + body_lines.len().to_string() + " lines)]\n"; set dropped_any=true. (d) Else emit each body line verbatim + '\n'.
  13. 4. pre_diff = leading_pre_diff_lines(content) (all lines before first 'diff --git', each with '\n' re-attached). If non-empty, prepend it to output (build a new string = pre_diff + output).
  14. 5. GATE: if !dropped_any OR output.len() >= content.len() -> Err(Skipped("diff_noise", "no droppable hunks")). (Never inflate.)
  15. 6. key = md5_hex_24(content) (MD5 of content bytes, lowercase hex, truncated to 24 chars). store.put(key, content) BEFORE returning. Append trailing marker: output += "\n[diff_noise CCR: hash=" + key + "]".
  16. 7. return OffloadOutput::from_lengths(content.len(), output, key).
  17. === parse_segments(content) ===
  18. Walk content.lines(); track current Option<Segment>, in_body bool. For each line: if starts_with("diff --git"): push current if any; start new Segment{new_path=parse_new_path(line), header_lines=[line], body_lines=[]}; in_body=false; continue. If no current segment yet (pre-diff prelude): continue (skip). If !in_body: if line.starts_with("@@") { in_body=true; body_lines.push(line); continue } else header_lines.push(line). Else body_lines.push(line). At EOF push final current. NOTE: the @@ hunk-header line itself goes into body_lines, not header_lines.
  19. === parse_new_path(header) === rfind(" b/") in the header; if found return header[idx+3..] (everything after the last ' b/'); else empty string.
  20. === leading_pre_diff_lines(content) === concatenate every line from start until (not including) first line starting with "diff --git", each line + '\n'.
  21. === is_lockfile(path) === if path empty -> false. For each suffix in lockfile_suffixes: if path.ends_with(suffix): prefix_len = path.len()-suffix.len(); if prefix_len==0 -> true (whole path IS the lockfile name); else check the byte immediately before the suffix: if it is '/' or '\\' -> true. (Path-segment-boundary match: 'crates/foo/Cargo.lock' matches, bare 'Cargo.lock' matches, 'MyCargo.lock' does NOT, 'FakeCargo.lockfile' does NOT since it doesn't end_with the suffix.)
  22. === Segment.body_is_whitespace_only() === adds=[], subs=[], saw_change=false. For each body line: match first byte: b'+' AND not starts_with("+++") -> saw_change=true, adds.push(strip_ws(line[1..])); b'-' AND not starts_with("---") -> saw_change=true, subs.push(strip_ws(line[1..])); else ignore (context, @@, etc). If !saw_change -> false (no changes at all is NOT whitespace-only). Return adds == subs (ORDER-AWARE vector equality — pairing of all + bodies vs all - bodies must be element-for-element equal after ws strip).
  23. === strip_ws(s) === filter out every ASCII-whitespace char (is_ascii_whitespace: space, \t, \n, \r, \x0b form-feed-vtab, \x0c) and collect remaining chars into a String. NOTE: only ASCII whitespace, not unicode whitespace.

  - diff_noise min_lines default is 30 (DIFFERENT from the 50 used by log/diff bloat configs) — easy to get wrong.
  - diff_noise: body_is_whitespace_only returns FALSE when there were no +/- change lines at all (saw_change guard) — a pure-context hunk is NOT whitespace-only and is NOT dropped.
  - diff_noise: whitespace comparison is ORDER-AWARE element-wise vector equality (adds == subs), and an unequal COUNT of + vs - lines automatically fails. Only ASCII whitespace stripped (is_ascii_whitespace), not Unicode spaces.
  - diff_noise: lockfile precedence — if both lockfile and whitespace conditions are true, reason emitted is "lockfile".
  - diff_noise: path-segment-aware matching prevents dropping user files like MyCargo.lock; suffix must sit at start-of-path or right after '/' or '\\'. 'FakeCargo.lockfile' fails because ends_with("Cargo.lock") is false.

  - diff_noise: header_lines (incl diff --git, index, mode, ---, +++) are ALWAYS emitted verbatim even for dropped files, so the LLM still sees the file changed; only the body (from @@ onward) is replaced by the cell marker.
  - diff_noise: leading pre-diff prelude (git format-patch commit message / Subject lines before first 'diff --git') is preserved verbatim and prepended.
  - diff_noise: the @@ hunk-header line is part of body_lines, so it IS counted in the 'N lines' dropped count and IS removed when a hunk is dropped.

-- publicApi --

-- algorithm --
  1. STAGE 0 — Early-return guard: `if text.is_empty() || !text.contains('<') { return (text.to_string(), Vec::new(), stats) }`. This is the only cheap structural sniff; do not do a full pass when there is no '<'.
  2. STAGE 0b — Collision detection / prefix selection: if `text.contains(DEFAULT_PREFIX)` (i.e. contains the literal "{{HEADROOM_TAG_"), enter the salt loop: for salt in 0u32..16 build `candidate = format!("{{{{HEADROOM_TAG_{salt}_")` (e.g. "{{HEADROOM_TAG_5_") and pick the first candidate NOT contained in text; set stats.placeholder_collision_avoided=true. If all 16 salts collide, use the fallback prefix literal "{{HEADROOM_TAG_FALLBACK_a4f1c7e2_" (still collision_avoided=true). If DEFAULT_PREFIX is absent, use DEFAULT_PREFIX and collision_avoided=false. The chosen `prefix` is used for ALL placeholders in this call.
  3. STAGE 1 — Single-pass byte walk to collect spans. Work on text.as_bytes(). cursor i=0; loop while i<n: if bytes[i] != b'<' advance to the next '<' (memchr-style skip) or to EOF; else call parse_tag_at(bytes, i).
  4. parse_tag_at(bytes, start) [bytes[start]==b'<']: j=start+1; if bytes[j]==b'/' set is_close=true and j+=1. Then require is_name_start(bytes[j]) (ASCII letter or '_'); if j>=len or not a name-start char => return NotTag. Scan name while is_name_cont(bytes[j]) (ASCII alphanumeric or one of '_','-','.',':'); name_end = first non-name-cont index. For CLOSE tags: skip whitespace, require b'>' => Close{name_end, tag_end=index after '>'}, else NotTag. For OPEN tags: lex attributes scanning forward; on '"' or '\'' scan to the matching quote (NotTag if EOF before close quote); detect "/>" => is_self_closing=true; on '>' => Open{name_end, tag_end, is_self_closing}; EOF before '>' => NotTag. Because '!' and '?' are NOT name-start chars, `<!-- -->`, `<!DOCTYPE>`, `<?xml?>`, CDATA all return NotTag and are emitted verbatim.
  5. After parse: on NotTag advance i by 1; otherwise advance i to tag_end (offset just past the closing '>').
  6. STAGE 1 classification per parsed tag: extract name_lower = lowercase of bytes[name_start..name_end]. Increment stats.tags_seen. If is_known_html_tag(name_lower) => HTML, do nothing (emit verbatim later, skip spans), increment stats.html_tags_skipped. Otherwise it is a CUSTOM tag, handled below.
  7. CUSTOM self-closing `<tag/>`: emit Span{start=open_start, end=tag_end, kind=SelfClosing} immediately; increment stats.self_closing_protected. (Same in both modes.)
  8. CUSTOM OPEN tag (not self-closing): in marker-only mode (compress_tagged_content=true) immediately push Span{start=i, end=tag_end, kind=OpenMarker} to spans, then push OpenTag{name_lower, open_start=i} to the stack. In block mode (compress_tagged_content=false) push only OpenTag{name_lower, open_start=i} to the stack (no span yet).
  9. CUSTOM CLOSE tag `</tag>`: search the stack from TOP backwards for an entry whose name_lower matches. If NO match => orphan close: emit verbatim (not added to spans), increment stats.orphan_closes, continue. If match at index k: In BLOCK mode — truncate the stack to k then pop the matched entry; REMOVE from the spans vec any spans whose start >= matched.open_start (these are inner nested self-closing spans now subsumed); emit one Span{start=matched.open_start, end=close.tag_end, kind=Block}; increment stats.custom_blocks_protected. In MARKER-ONLY mode — truncate stack to k and pop; emit a separate Span{start=close_start, end=close.tag_end, kind=CloseMarker} (the OpenMarker span was already emitted at open time).
  10. END OF WALK: unmatched OpenTags remain on the stack. In block mode they were never turned into spans, so their content is NOT protected (emitted verbatim). In marker-only mode their OpenMarker spans were already pushed and ARE KEPT (still protected). Inner self-closing spans that were not subsumed by a collapsed outer Block are kept.
  11. STAGE 2 — emit_output via OFFSET SLICING (NOT strings.Replace). spans are already sorted left-to-right (monotonic walk) and non-overlapping (block mode collapses nested spans into the outer Block; marker mode yields disjoint open/close ranges). cursor=0; for (counter, span) in spans.enumerate(): out += text[cursor..span.start]; placeholder = format!("{prefix}{counter}{PLACEHOLDER_SUFFIX}") (counter is u64, always appended after the possibly-salted prefix and before "}}"); push (placeholder, text[span.start..span.end]) to blocks; out += placeholder; cursor = span.end. After loop: out += text[cursor..] (the tail). Return (out, blocks, stats).
  12. RESTORE — restore_tags(text, blocks) => restore_tags_with_request_id(text, blocks, None). working = text.to_string(); forward-iterate blocks in order: for (placeholder, original): if working.contains(placeholder) => working = working.replace(placeholder, original) (replace ALL occurrences — safe because placeholders are unique per call); else => emit ERROR-level structured log (include request_id if present) and SKIP this block (do not inject the original, do not append — 'Hotfix-A9' discards the lost wrap). Return working.

  * PLACEHOLDER_SUFFIX = }}

-- edgeCases --
  - Early-return guard MUST come first: empty text OR no '<' => return (text, empty blocks, default stats). Avoids any pass.
  - Offset slicing, NOT strings.Replace: build output by copying text[cursor..span.start] then placeholder, advancing cursor=span.end, finally append the tail text[cursor..]. This deliberately avoids the Python 'first-occurrence / repeated-substring' replace bug.
  - Spans are guaranteed sorted (monotonic forward walk) and non-overlapping (block mode collapses nested inner spans into the outer Block by removing spans with start>=open_start; marker mode yields disjoint open/close byte ranges). Do not re-sort or you may diverge; preserve discovery order = byte order.
  - HTML5 tags are matched case-insensitively (lowercase the name before HashSet lookup) and are emitted verbatim — never protected, never added to spans, only counted in stats.html_tags_skipped.
  - Custom-tag name matching for open/close is by name_lower (case-insensitive). Close search scans the stack from TOP backwards for the matching name.
  - Orphan close (no matching open on stack): emit verbatim, increment stats.orphan_closes, continue — do NOT protect it.
  - Orphan open (no close before EOF): block mode => content NOT protected (no span ever created), emitted verbatim. Marker-only mode => its already-pushed OpenMarker span is KEPT (still protected).
  - Nested custom tags in block mode collapse to the OUTERMOST block: when a close matches at stack index k, truncate stack to k and remove any spans whose start >= matched.open_start before pushing the single outer Block span.
  - Self-closing custom `<tag/>` always protected as SelfClosing in BOTH modes; counted in self_closing_protected. A self-closing span nested inside an unmatched outer open survives (not collapsed).
  - compress_tagged_content semantics: false (default) = protect whole <tag>...</tag> Block, hiding inner content from the compressor. true = protect only OpenMarker + CloseMarker, letting inner content flow to the compressor (open and close become separate blocks entries).
  - Non-tag '<' constructs — `<!-- comment -->`, `<!DOCTYPE ...>`, `<?xml ...?>`, CDATA — all return NotTag (because '!','?' are not is_name_start) and are emitted verbatim; cursor advances by 1 past the '<'.
  - Open-tag attribute lexer must honor quoted strings (single and double quotes): inside a quote, '>' does not terminate the tag; EOF inside a quote => NotTag. Self-closing detected via '/>' before '>'.
  - Restore is forward-iteration with replace-ALL (placeholders unique per call). Missing placeholder => log ERROR (include request_id if provided) and SKIP the block; never inject the original or append (Hotfix-A9). Restore must be exact inverse so round-trip protect->compress->restore is lossless for surviving placeholders.
  - Byte-offset scanning operates on bytes (text.as_bytes()); name char classes are ASCII-only predicates. Multi-byte UTF-8 inside content/attributes is copied through unchanged via offset slices (slice boundaries always fall on tag-structure ASCII bytes '<','>','/','"',quote, so they are valid UTF-8 boundaries).
  - stats.placeholder_collision_avoided is set true whenever a non-default (salted or fallback) prefix is chosen.

ccrMarker: n/a — this file is a sizing/k-selection helper, not an offload transform; it emits no CCR marker and has no min_compression_ratio_for_ccr gate. (The only compression usage here is zlib measurement for the k-boost heuristic.)

-- publicApi --
  * compute_optimal_k: pub fn compute_optimal_k(items: &[&str], bias: f64, min_k: usize, max_k: Option<usize>) -> usize
      Top-level entry. Returns how many leading items/rows/lines to keep. 3-tier: fast-path (n<=8 keep all), near-total-redundancy (<=3 simhash clusters), then Kneedle-on-bigram-curve + diversity floor + bias multiplier + zlib validation, all clamped to [min_k, effective_max].
  * simhash: pub fn simhash(text: &str) -> u64
      64-bit SimHash fingerprint. Lowercases text, slides a 4-char window, MD5-hashes each window, takes first 8 digest bytes big-endian as u64, votes each of 64 bits (+1 if set, -1 if clear), final bit set when vote>0.
  * hamming_distance: #[inline] pub fn hamming_distance(a: u64, b: u64) -> u32
      Returns (a ^ b).count_ones() — number of differing bits between two fingerprints.
  * count_unique_simhash: pub fn count_unique_simhash(items: &[&str], threshold: u32) -> usize
      Greedy single-pass clustering: for each item's fingerprint, if it is within `threshold` Hamming distance of any existing cluster representative it joins that cluster (no new cluster), else becomes a new representative. Returns number of clusters (distinct groups). Returns 0 for empty input.
  * compute_unique_bigram_curve: pub fn compute_unique_bigram_curve(items: &[&str]) -> Vec<usize>
      Builds cumulative-unique-bigram coverage curve. For each item (in order): lowercase, split on whitespace; if <2 words synthesize a (first_word_or_empty, "") pseudo-bigram, else insert all adjacent word pairs into a global HashSet; push current set size. curve[i] = cumulative distinct bigrams after item i.
  * find_knee: pub fn find_knee(curve: &[usize]) -> Option<usize>
      Kneedle knee detection. Normalizes index and value to [0,1], finds the point of max (y_norm - x_norm) deviation from the diagonal; returns knee_idx+1. Returns None if curve len < 3 or max deviation < 0.05; returns Some(1) if curve is flat (y_max==y_min).
  * validate_with_zlib: pub fn validate_with_zlib(items: &[&str], k: usize, max_k: usize, tolerance: f64) -> usize
      Compression-ratio sanity check. If the k-subset compresses much better than the full set (|full_ratio - subset_ratio| > tolerance), bumps k by 20% (k*1.2), capped at max_k. Skips (returns k unchanged) when k>=len, k>=max_k, or full_text.len()<200.
  * zlib_compressed_len: fn zlib_compressed_len(bytes: &[u8]) -> usize
      Private helper. Returns the byte length of `bytes` zlib-compressed at Compression::fast() (flate2 level 1).

-- algorithm --
  1. compute_optimal_k(items, bias, min_k, max_k): let n = items.len(); let effective_max = max_k.unwrap_or(n).
  2. Tier 1 fast path: if n <= 8 return n (keep everything; no clamp applied here, returns raw n).
  3. Near-total redundancy check: let unique_count = count_unique_simhash(items, 3) (Hamming threshold literal 3). If unique_count <= 3: let k = min_k.max(unique_count); return k.min(effective_max). (i.e. clamp unique_count up to min_k then down to effective_max).
  4. Tier 2 Kneedle: let curve = compute_unique_bigram_curve(items); let mut knee = find_knee(&curve).
  5. Compute diversity_ratio = unique_count as f64 / n as f64.
  6. Adjust knee via match: (a) None branch (no saturation found): keep_fraction = 0.3 + 0.7 * diversity_ratio; knee = Some(min_k.max((n as f64 * keep_fraction) as usize)). (b) Some(k) when diversity_ratio > 0.7: floor = min_k.max((n as f64 * (0.3 + 0.7 * diversity_ratio)) as usize); knee = Some(k.max(floor)) — high-diversity floor so we never drop below n*(0.3+0.7*div). (c) otherwise: keep knee as-is.
  7. Defensive unwrap: let knee = knee.unwrap_or(min_k) (knee paths above always set Some).
  8. Apply bias multiplier (Python int(knee*bias)): let mut k = min_k.max((knee as f64 * bias) as usize); then k = k.min(effective_max).
  9. Tier 3 zlib validation: k = validate_with_zlib(items, k, effective_max, 0.15).
  10. Final clamp: return min_k.max(k.min(effective_max)).
  11. --- simhash(text): lower = text.to_lowercase(); chars = lower.chars().collect(); n = chars.len(); iter_count = if n <= 3 { 1 } else { n - 3 }; votes = [0i32; 64]; for i in 0..iter_count { gram = chars.iter().skip(i).take(4).collect::<String>(); digest = Md5::digest(gram.as_bytes()); h = u64::from_be_bytes(digest[0..8]); for j in 0..64 { if (h >> j) & 1 == 1 { votes[j]+=1 } else { votes[j]-=1 } } } fingerprint=0; for j in 0..64 { if votes[j] > 0 { fingerprint |= 1<<j } } return fingerprint.
  12. --- count_unique_simhash(items, threshold): if empty return 0; fingerprints = items.map(simhash); clusters = Vec::new(); for fp in fingerprints { matched=false; for rep in &clusters { if hamming_distance(fp, rep) <= threshold { matched=true; break } } if !matched { clusters.push(fp) } } return clusters.len(). Note: greedy, order-dependent, compares only against representatives (first-seen of each cluster).
  13. --- compute_unique_bigram_curve(items): seen: HashSet<(String,String)>; curve = Vec; for item in items { lower=item.to_lowercase(); words = lower.split_whitespace().collect(); if words.len() < 2 { first = words.first().copied().unwrap_or(""); seen.insert((first.to_string(), String::new())) } else { for j in 0..words.len()-1 { seen.insert((words[j].to_string(), words[j+1].to_string())) } } curve.push(seen.len()) } return curve.
  14. --- find_knee(curve): n = curve.len(); if n < 3 return None; x_min=0; x_max=n-1; y_min=curve[0] as f64; y_max=curve[n-1] as f64; if (y_max-y_min).abs() < f64::EPSILON return Some(1); x_range=(x_max-x_min) as f64; y_range=y_max-y_min; max_diff=-1.0; knee_idx=None; for (i,&y) in curve.enumerate() { x_norm=(i-x_min) as f64 / x_range; y_norm=(y as f64 - y_min)/y_range; diff = y_norm - x_norm; if diff > max_diff { max_diff=diff; knee_idx=Some(i) } } if max_diff < 0.05 return None; return knee_idx.map(|i| i+1).
  15. --- validate_with_zlib(items, k, max_k, tolerance): if k >= items.len() || k >= max_k return k; full_text = items.join("\n"); subset_text = items[..k].join("\n"); if full_text.len() < 200 return k; full_compressed = zlib_compressed_len(full_text.as_bytes()); subset_compressed = zlib_compressed_len(subset_text.as_bytes()); full_ratio = if !full_text.is_empty() { full_compressed/full_text.len() } else { 1.0 }; subset_ratio = if !subset_text.is_empty() { subset_compressed/subset_text.len() } else { 1.0 }; ratio_diff = (full_ratio - subset_ratio).abs(); if ratio_diff > tolerance { adjusted = (k as f64 * 1.2) as usize; return adjusted.min(max_k) } return k.
  16. --- zlib_compressed_len(bytes): ZlibEncoder::new(Vec::new(), Compression::fast()) (flate2 level 1); write_all(bytes); finish(); return compressed.len().

-- patterns --
  * simhash_lowercase = text.to_lowercase()
      (Unicode-aware lowercasing before 4-gram windowing in simhash; must use rune-aware lowering, not ASCII-only, for parity.)
  * simhash_hash = Md5::digest(gram.as_bytes()) -> first 8 bytes big-endian as u64
      (MD5 of each 4-char gram's UTF-8 bytes; take digest[0..8] as big-endian u64 (mirrors Python int(hexdigest[:16], 16)).)
  * bigram_split = lower.split_whitespace()
      (whitespace tokenization (collapses runs of whitespace, trims) for bigram curve; Go equivalent strings.Fields.)
  * join_separator = items.join("\n")
      (items joined with single newline to form full_text/subset_text for zlib measurement.)
  * zlib_encoder = ZlibEncoder::new(Vec::new(), Compression::fast())
      (zlib (RFC 1950) stream, compression level 1 (fast); Go equivalent: compress/zlib with flate level 1 (flate.BestSpeed). Compressed LENGTH only is used, never the bytes.)

-- thresholds --
  * fast_path_n = 8  :: if n <= 8, compute_optimal_k returns n immediately (keep all items, no clamp).
  * redundancy_cluster_count = 3  :: if count_unique_simhash(items, 3) yields unique_count <= 3, return clamp(unique_count) — near-total redundancy short-circuit.
  * simhash_hamming_threshold = 3  :: Hamming-distance threshold literally passed to count_unique_simhash inside compute_optimal_k for clustering similar items (<= threshold means same cluster).
  * simhash_ngram_size = 4  :: 4-character sliding window (4-gram) used per hash iteration in simhash.
  * simhash_iter_count_short = 1  :: when char-count n <= 3, iterate exactly once over the whole (short) string (mirrors Python range(max(1, n-3))).
  * simhash_iter_count_long = n - 3  :: when n >= 4, number of 4-gram windows is n-3.
  * simhash_bits = 64  :: fingerprint width; votes array length 64; uses first 8 bytes of MD5 digest big-endian.
  * keep_fraction_base = 0.3  :: base keep fraction when diversity_ratio is 0 (keep ~30%). keep_fraction = 0.3 + 0.7 * diversity_ratio.
  * keep_fraction_diversity_mult = 0.7  :: diversity multiplier; at diversity_ratio=1.0 keep_fraction=1.0 (keep 100%).
  * diversity_floor_gate = 0.7  :: when a knee IS found and diversity_ratio > 0.7, apply a floor min_k.max(n*(0.3+0.7*div)) so knee is not allowed to drop below it (knee = k.max(floor)).
  * kneedle_deviation_threshold = 0.05  :: in find_knee, if max (y_norm - x_norm) deviation < 0.05, return None (no knee). Strict < comparison.
  * kneedle_min_curve_len = 3  :: find_knee returns None when curve.len() < 3.
  * kneedle_flat_curve_return = 1  :: if y_max - y_min is ~0 (|.| < f64::EPSILON), find_knee returns Some(1) (Python literal 1).
  * kneedle_knee_offset = +1  :: find_knee returns knee_idx + 1 (1-based count of items to keep, not 0-based index).
  * bigram_min_words = 2  :: items with < 2 whitespace-split words synthesize a (first_word_or_empty, "") pseudo-bigram; >= 2 words insert all adjacent pairs.
  * zlib_tolerance = 0.15  :: tolerance literal passed to validate_with_zlib from compute_optimal_k; if |full_ratio - subset_ratio| > 0.15 the subset compresses too well so bump k.
  * zlib_boost_factor = 1.2  :: when ratio_diff > tolerance, k is increased 20%: adjusted = (k*1.2) as usize, then min(max_k).
  * zlib_min_content_len = 200  :: skip zlib validation entirely (return k unchanged) when full_text.len() < 200 bytes (zlib overhead dominates).
  * zlib_compression_level = Compression::fast() (flate2 level 1)  :: compression level used by zlib_compressed_len for both full and subset measurement.
  * empty_ratio_default = 1.0  :: full_ratio/subset_ratio default to 1.0 when their text is empty, in validate_with_zlib.

-- edgeCases --
  - Fast path returns raw n WITHOUT clamping to min_k/effective_max — only path that bypasses the final clamp. For n<=8 the function returns n even if max_k < n. (Faithful port must replicate: do NOT clamp the n<=8 return.)
  - Redundancy branch (unique_count<=3) clamps: min_k.max(unique_count).min(effective_max).
  - count_unique_simhash returns 0 for empty items slice (cannot reach here since n<=8 fast path already handles small/empty input).
  - simhash short-string handling: n<=3 chars iterates ONCE over whole string (skip(0).take(4) collects <=3 chars); do not skip hashing short strings.
  - simhash uses to_lowercase() (Unicode-aware lowercasing) and .chars() (Unicode scalar values, NOT bytes) for the 4-gram window — must operate on chars/runes, not raw bytes, for parity.
  - simhash bit extraction is (h >> j) & 1 for j in 0..64 where h is first-8-MD5-bytes big-endian; vote +1 if bit set else -1; final bit set only when vote strictly > 0 (ties / zero votes => bit 0).
  - find_knee: curve.len() < 3 => None; flat curve (y_max==y_min within EPSILON) => Some(1); otherwise needs max deviation >= 0.05 (strict < 0.05 => None).
  - find_knee returns 1-based count (knee_idx + 1), not the index itself.
  - Diversity floor only applies when knee is Some AND diversity_ratio > 0.7 (strict >).
  - None-knee branch sets knee from keep_fraction scaling; defensive unwrap_or(min_k) should never actually fire because both reachable match arms produce Some.
  - validate_with_zlib guards: early-return k unchanged if k>=items.len() OR k>=max_k OR full_text.len()<200. Items joined with '\n' (newline) for both full and subset text. Subset is items[..k] (first k items, leading prefix — ORDER MATTERS, keeps prefix).
  - bias multiplier truncates toward zero ((knee as f64 * bias) as usize); all f64->usize casts are truncation (floor for positives), matching Rust `as` semantics and Python int().
  - No content-type-specific overrides exist in this file: keep_fraction is computed dynamically from diversity_ratio, not a per-content-type constant. min_k/max_k/bias are caller-supplied, not defaulted here.
  - Final clamp is min_k.max(k.min(effective_max)) — clamp DOWN to effective_max first, then UP to min_k (min_k wins if min_k > effective_max).

-- algorithm --
  1. SCORING CONTRACT: every detector's score(line, ctx) returns an ImportanceSignal { category: Option, priority f32 in [0,1], confidence f32 in [0,1] }. No-match = neutral() = {None, 0.0, 0.0}. priority semantics: 0.0 = drop first, 1.0 = keep at all costs. There is NO combined_score helper; priority and confidence stay separate. KeywordDetector emits a fixed confidence of KEYWORD_CONFIDENCE=0.7 on every match and uses the per-category priority constant as the priority.
  2. KeywordRegistry::default_set() builds keyword lists per category and compiles aho-corasick automatons. The Error and Importance lists are combined into the 'universal' automaton; Warning into the 'warning' automaton; Security into the 'security' automaton; the error-indicator list into the substring-only 'indicators' automaton.
  3. AUTOMATON CONFIG (all word-boundary automatons): build aho-corasick with ascii_case_insensitive(true) and match_kind(LeftmostLongest). Iterate matches with find_iter(). For each raw match, validate WORD BOUNDARIES: left boundary OK if match start == 0 OR the byte immediately before is a non-word byte; right boundary OK if match end == line.len() OR the byte immediately after is a non-word byte. Word byte = one of [A-Za-z0-9_]. Reject matches failing either boundary. (Go: scan bytes; treat ASCII letters/digits/underscore as word chars.)
  4. KeywordDetector::score(line, ctx) STAGE ORDER (returns the single highest-priority category that is allowed in this context; this is a max-by-priority pick across the gated categories, NOT a sum): 1) Run the universal automaton (Error+Importance) with word-boundary validation — fires in ALL contexts. 2) Run warning automaton with word-boundary validation — fires ONLY in Search, Log, Text (EXCLUDED from Diff). 3) Run security automaton with word-boundary validation — fires ONLY in Diff context. 4) If ctx == Text, also check Markdown prefixes (prefix match on the line, NOT word-boundary, NOT via aho-corasick): if line starts with any markdown prefix, that is a Markdown match.
  5. PRIORITY SELECTION: among the categories that matched AND are allowed in the current context, return the one with the highest per-category priority constant. Map category -> priority: Error=ERROR_PRIORITY(0.95), Security=SECURITY_PRIORITY(0.85), Warning=WARNING_PRIORITY(0.75), Importance=IMPORTANCE_PRIORITY(0.6), Markdown=MARKDOWN_PRIORITY(0.45). Return ImportanceSignal::matched(winningCategory, winningPriority, 0.7). If nothing matched, return neutral() {None, 0.0, 0.0}. Effective output priority range: 0.0 (no match) or one of {0.45,0.6,0.75,0.85,0.95}; confidence always 0.0 (no match) or 0.7 (match).
  6. Note the deliberate context split: a Diff line uses Security keywords (auth/password/secret in added/removed code is interesting) but NOT Warning; Search/Log/Text use Warning but NOT Security. Error+Importance are universal. Markdown only matters in plain Text.
  7. contains_error_indicator(line): SUBSTRING-only match (no word boundaries) against the error-indicator set {error, fail, exception, traceback, fatal, panic, crash} using a separate aho-corasick (case-insensitive). Returns bool. This is the cheap 'error-shaped' triage; note it includes 'traceback' but OMITS the four error extras (abort/timeout/denied/rejected) that the full error word-boundary set includes.
  8. Tiered::score(line, ctx) COMBINATION = short-circuit escalation, ordered, NOT sum/max-priority/weighted-average. Init best = neutral(). For each tier in insertion order: signal = tier.score(line, ctx); IF signal.confidence >= ESCALATE_THRESHOLD(0.7) RETURN signal immediately (first confident tier wins); ELSE if signal.confidence > best.confidence then best = signal. After all tiers, return best. So the result is the first tier to reach confidence>=0.7, otherwise the highest-confidence signal seen. Comparison/escalation is on CONFIDENCE, not priority.
  9. GO PORT MVP: implement ImportanceCategory/ImportanceContext as enums; ImportanceSignal as a struct with a Category field that can be absent (pointer or a HasMatch bool + Category value). Implement the aho-corasick automatons either via a real Go aho-corasick or, for a zero-dep MVP, via case-insensitive substring scan + the word-boundary validation rule (LeftmostLongest only matters when one keyword is a prefix of another; in the default lists 'warn' is a prefix of 'warning' and 'fail' a prefix of 'failed'/'failure' — prefer the LONGEST match at a given start to mirror LeftmostLongest, though for category selection it doesn't change the winning category). Match case-insensitively on ASCII.

-- patterns --
  * error_keywords = error, exception, fail, failed, failure, fatal, critical, crash, panic, abort, timeout, denied, rejected
      (Error category list (13 keywords, in this order). Combined with importance into the 'universal' automaton; fires in ALL contexts. Word-boundary, case-insensitive. abort/timeout/denied/rejected were added (had been missing from the old Python regex).)
  * warning_keywords = warn, warning
      (Warning category list (2 keywords). Fires ONLY in Search, Log, Text (NOT Diff). Word-boundary, case-insensitive. 'warn' is a prefix of 'warning' so LeftmostLongest matters for span but not for category.)
  * importance_keywords = important, note, todo, fixme, hack, xxx, bug, fix
      (Importance category list (8 keywords). Combined with error into 'universal'; fires in ALL contexts. Word-boundary, case-insensitive.)
  * security_keywords = security, auth, password, secret
      (Security category list (4 keywords). Fires ONLY in Diff context. Word-boundary, case-insensitive. NOTE: 'token' was deliberately REMOVED to avoid false positives on LLM token metrics like input_tokens=512.)
  * markdown_prefixes = "# ", "## ", "### ", "#### ", "**", "> "
      (Markdown prefix patterns (6). Checked ONLY in Text context, via line-PREFIX match (line starts-with), NOT aho-corasick, NOT word-boundary. Any hit yields the Markdown category.)
  * error_indicators = error, fail, exception, traceback, fatal, panic, crash
      (Substring-only triage set (7 keywords) for contains_error_indicator(). Matched WITHOUT word boundaries (substring), case-insensitive. Includes 'traceback'; omits abort/timeout/denied/rejected. Separate from the full error word-boundary set.)
  * word_byte_class = [A-Za-z0-9_]
      (Definition of a 'word byte' for the boundary post-filter. A keyword match is valid only if the byte before its start and the byte after its end are NOT word bytes (or at string edges).)

-- thresholds --
  * KEYWORD_CONFIDENCE = 0.7  :: Fixed confidence emitted by KeywordDetector on every match (the confidence field of the returned ImportanceSignal). Equal to ESCALATE_THRESHOLD so a keyword match always escalates in a Tiered chain.
  * ERROR_PRIORITY = 0.95  :: Priority value used when the winning category is Error. Highest of the five category priorities.
  * SECURITY_PRIORITY = 0.85  :: Priority value used when the winning category is Security (Diff context only).
  * WARNING_PRIORITY = 0.75  :: Priority value used when the winning category is Warning (Search/Log/Text only).
  * IMPORTANCE_PRIORITY = 0.6  :: Priority value used when the winning category is Importance.
  * MARKDOWN_PRIORITY = 0.45  :: Priority value used when the winning category is Markdown (Text context only). Lowest category priority.
  * ESCALATE_THRESHOLD = 0.7  :: Tiered combinator threshold: if a tier's signal.confidence >= 0.7, Tiered returns that signal immediately (short-circuit). Comparison is on confidence, not priority.
  * neutral_priority = 0.0  :: priority of the no-match (neutral) signal; also the floor of the priority range. Semantically 'drop first'.
  * neutral_confidence = 0.0  :: confidence of the no-match (neutral) signal; floor of confidence range.
  * max_priority = 1.0  :: Top of the priority range, semantically 'keep at all costs'. Not emitted by KeywordDetector (max it emits is 0.95) but is the trait contract ceiling.
  * max_confidence = 1.0  :: Top of the confidence range (detector certainty). KeywordDetector emits at most 0.7.

-- edgeCases --
  - Context gating is hard: Warning keywords are SUPPRESSED in Diff context; Security keywords ONLY fire in Diff context; Markdown ONLY in Text context. Error+Importance fire in all contexts. Getting these gates wrong changes which category wins.
  - 'token' must NOT be in the Security list (removed to stop false positives on LLM token-count strings like input_tokens=512). Do not re-add it.
  - Word-boundary validation is required for all aho-corasick category matches: reject a match if the preceding or following byte is a word byte ([A-Za-z0-9_]); start==0 and end==len count as valid boundaries. This prevents e.g. 'failover' or 'noteworthy'/'footnote' from matching 'fail'/'note' substrings.
  - contains_error_indicator() is intentionally SUBSTRING-only (no boundary check) and uses a DIFFERENT, smaller list (includes traceback, omits abort/timeout/denied/rejected). Do not reuse the full error set or apply word boundaries here.
  - Markdown prefixes are matched as line-START prefixes only (and only in Text), NOT via the automaton and NOT word-boundary; '**' and '> ' etc. They are checked after the automaton categories.
  - Automaton is ascii_case_insensitive — matching is ASCII case-folding only; non-ASCII bytes are treated as word/non-word per the byte class and are never lowercased. Scan by bytes, not runes, to mirror Rust byte indexing for boundary checks.
  - match_kind = LeftmostLongest: where one keyword is a prefix of another (warn/warning, fail/failed/failure), prefer the longest match starting at a position. This affects the matched span but not the winning category (both map to the same category).
  - When multiple categories match on one line, return the single one with the HIGHEST per-category priority (max-by-priority), not a sum and not the first found. Priority order: Error(0.95) > Security(0.85) > Warning(0.75) > Importance(0.6) > Markdown(0.45).
  - Tiered escalation compares CONFIDENCE (>= 0.7), not priority. Because KeywordDetector always emits confidence 0.7 on a match, any keyword match short-circuits a Tiered chain immediately. A no-match (confidence 0.0) never escalates and Tiered falls through to the best (highest-confidence) signal, defaulting to neutral() if all tiers miss.
  - Never invent a combined_score: the codebase keeps priority and confidence separate; there is no helper merging them. Downstream consumers read both fields.

  5. === bm25.rs TOKENIZER ===

TOKEN_PATTERN is a single regex with 3 alternatives in this exact order (order is critical — UUID before digits before generic alnum so a UUID is captured whole, and a 4+ digit run is captured before the generic alnum rule could split it): UUID | numeric-id(\b\d{4,}\b) | [a-zA-Z0-9_]+. Lowercasing happens BEFORE matching (so hex letters in UUIDs become lowercase). No stopwords, no stemming; punctuation is implicitly dropped (only matched substrings are kept).
  7. === bm25.rs bm25_score(doc_tokens, query_freq, avg_doc_len) -> (f64, Vec<String>) ===
  8. Guard: if doc_tokens.is_empty() OR query_freq.is_empty() return (0.0, []).
  9. Build doc_freq: HashMap<&str, usize>; for t in doc_tokens { *doc_freq.entry(t.as_str()).or_insert(0) += 1 }. doc_len = doc_tokens.len() as f64.
  10. Resolve effective avg doc length: avgdl = if avg_doc_len > 0.0 { avg_doc_len } else if doc_len > 0.0 { doc_len } else { 1.0 }. (In single-doc score() avg_doc_len is passed 0.0 so avgdl collapses to doc_len => length-normalization term becomes neutral.)
  11. idf = 2.0_f64.ln() (≈0.6931, a fixed constant — NOT ln(N/df); single-doc simplified IDF).
  12. Iterate query terms in SORTED order: let mut keys: Vec<&String> = query_freq.keys().collect(); keys.sort(); for term in keys { let qf = query_freq[term]; let Some(&f) = doc_freq.get(term.as_str()) else { continue }; let f = f as f64; numerator = f * (k1 + 1.0); denominator = f + k1 * (1.0 - b + b * doc_len / avgdl); term_score = idf * numerator / denominator; score += term_score * qf as f64; matched.push(term.clone()); }
  13. Return (score, matched). matched is exactly the query terms (alphabetically sorted) that appear in the doc, in sorted order.
  14. === bm25.rs finalize (shared shape for score & score_batch) ===
  15. Given (raw, matched): normalized = if normalize_score { (raw / max_score).min(1.0) } else { raw }. Then long-token bonus: if matched.iter().any(|t| t.len() >= 8) { normalized = (normalized + 0.3).min(1.0) }. (RelevanceScore::new also re-clamps to [0,1].)
  16. === bm25.rs score(item, context) ===
  17. item_tokens = tokenize(item); context_tokens = tokenize(context); query_freq = freq map (HashMap<String,usize>) over context_tokens. (raw, matched) = bm25_score(&item_tokens, &query_freq, 0.0). Apply finalize -> normalized.
  18. Build reason: n = matched.len(); if n==0 reason="BM25: no term matches"; if n==1 reason=format!("BM25: matched '{}'", matched[0]); if n>1: preview: Vec<&str> = matched.iter().take(3).map(|s| s.as_str()).collect(); suffix = if n > 3 { "..." } else { "" }; reason = format!("BM25: matched {} terms ({}{})", n, preview.join(", "), suffix).
  19. matched_capped = matched.iter().take(10).cloned().collect(). Return RelevanceScore::new(normalized, reason, matched_capped).
  20. === bm25.rs score_batch(items, context) ===
  21. context_tokens = tokenize(context). If context_tokens empty: return one RelevanceScore::empty("BM25: empty context") per item (all zero). query_freq = freq map over context_tokens.
  22. Pre-tokenize all items into Vec<Vec<String>>. total_len = sum of all item token counts. avg_len = total_len as f64 / items.len().max(1) as f64. (Computed once.)
  23. For each item_tokens: (raw, matched) = bm25_score(&item_tokens, &query_freq, avg_len); finalize -> normalized. reason: n=matched.len(); if n==0 reason="BM25: no matches" else reason=format!("BM25: {} terms", n). matched_capped = matched.iter().take(5).cloned().collect(). Push RelevanceScore::new(normalized, reason, matched_capped).
  24. === hybrid.rs compute_alpha(context) ===
  25. if !adaptive return base_alpha. context_lower = context.to_lowercase(). uuid_count = UUID_PATTERN.find_iter(context).count() [ORIGINAL case]; id_count = NUMERIC_ID_PATTERN.find_iter(context).count() [ORIGINAL case]; hostname_count = HOSTNAME_PATTERN.find_iter(&context_lower).count() [LOWERCASED]; email_count = EMAIL_PATTERN.find_iter(&context_lower).count() [LOWERCASED]. NOTE the case asymmetry: UUID/ID scan original text, hostname/email scan lowercased text.
  26. alpha = base_alpha; then a single if/else-if ladder (only ONE branch fires): if uuid_count>0 { alpha = alpha.max(0.85) } else if id_count>=2 { alpha = alpha.max(0.75) } else if id_count==1 { alpha = alpha.max(0.65) } else if hostname_count>0 || email_count>0 { alpha = alpha.max(0.6) }. Return alpha.clamp(0.3, 0.9).
  27. === hybrid.rs boost_bm25_only(bm25_result) ===
  28. let mut boosted = bm25_result.score; if !matched_terms.is_empty() { boosted = boosted.max(0.3); if matched_terms.len() >= 2 { boosted = (boosted + 0.2).min(1.0) } }. Return RelevanceScore::new(boosted, format!("Hybrid (BM25 only, boosted): {}", bm25_result.reason), bm25_result.matched_terms.clone()).
  29. === hybrid.rs score(item, context) ===
  30. bm25_result = bm25.score(item, context). if !embedding_available return boost_bm25_only(&bm25_result) [THE v0 PATH — always taken since embedding stubbed]. Else (future): emb = embedding.score(item, context); alpha = compute_alpha(context); combined = alpha*bm25_result.score + (1-alpha)*emb.score; reason = format!("Hybrid (\u{3b1}={:.2}): BM25={:.2}, Semantic={:.2}", alpha, bm25.score, emb.score); return new(combined, reason, bm25_result.matched_terms).
  31. === hybrid.rs score_batch(items, context) ===
  32. if items.is_empty() return []. bm25_results = bm25.score_batch(items, context). if !embedding_available return bm25_results.iter().map(|r| boost_bm25_only(r)).collect() [v0 PATH]. Else (future): emb_results = embedding.score_batch; alpha = compute_alpha(context); zip bm25 with emb, combined = alpha*bm25.score+(1-alpha)*emb.score, reason format "Hybrid (\u{3b1}={:.2}): BM25={:.2}, Emb={:.2}".
  33. === mod.rs create_scorer(tier) ===
  34. match tier.to_lowercase(): "bm25"=>Ok(BM25Scorer::default()); "hybrid"=>Ok(HybridScorer::default()); "embedding"=> let s=EmbeddingScorer::default(); if s.is_available() Ok(s) else Err("EmbeddingScorer requires the ONNX backend (not yet implemented in Rust)"); other=>Err(format!("Unknown scorer tier: {}. Valid tiers: bm25, embedding, hybrid", other)).

-- edgeCases --
  - RelevanceScore.new ALWAYS clamps score to [0.0,1.0] (mirrors Python __post_init__) — every code path re-clamps, so over/under-flow is impossible at the boundary.
  - bm25_score returns (0.0, []) immediately if doc_tokens empty OR query_freq empty — never proceeds to division.
  - avgdl fallback ladder (avg_doc_len>0 ? avg_doc_len : doc_len>0 ? doc_len : 1.0) prevents division-by-zero in the BM25 denominator. In single-doc score(), avg_doc_len is passed 0.0 deliberately so avgdl == doc_len, making the length-norm factor (1 - b + b*doc_len/avgdl) = 1.0 exactly (neutral).
  - score_batch with empty context (no tokens): returns one zero-score RelevanceScore::empty("BM25: empty context") per item — does NOT skip or shorten the output vec.
  - score_batch divides total_len by items.len().max(1) — guards against empty items (though if items is empty the empty-context branch may not trigger; hybrid's score_batch separately early-returns [] when items.is_empty()).
  - hybrid score_batch early-returns empty Vec when items.is_empty() (checked before tokenizing).
  - Long-token bonus uses Rust str.len() = BYTE length (not char count). For ASCII tokens (the regex only matches ASCII hex/digits/[a-zA-Z0-9_] plus dashes in UUID) byte len == char len, so this is safe for matched terms; a Go port must use len([]byte) / len(string) NOT utf8.RuneCountInString to match. (UUID tokens are 36 bytes, always >=8.)
  - Tokenizer lowercases BEFORE matching, so UUID hex letters and all words are lowercased; matched_terms are therefore lowercase.
  - Regex alternation order in TOKEN_PATTERN is load-bearing: UUID first (else it'd be split into alnum chunks at dashes), then \b\d{4,}\b, then [a-zA-Z0-9_]+. A leftmost-longest vs leftmost-first engine difference could change tokenization — Rust `regex` is leftmost-first within alternation at a given start position, matching Python `re`. Go's regexp is leftmost-longest (RE2/POSIX-ish for the overall match but leftmost-first for alternation in default mode — Go uses leftmost-first/Perl semantics by default, NOT POSIX). Verify Go regexp default (non-POSIX) gives same tokens; it should since it's Perl-like leftmost-first.
  - compute_alpha case asymmetry: UUID and numeric-ID counts scan the ORIGINAL context string; hostname and email counts scan the LOWERCASED string. Must replicate exactly — scanning the wrong case can change counts (e.g. uppercase hex UUIDs still match UUID_PATTERN which is case-insensitive in its class, so original-vs-lower is immaterial for UUID; but matters conceptually).
  - compute_alpha is an if/else-if ladder: exactly ONE tier fires (UUID dominates; then >=2 IDs; then 1 ID; then host/email). alpha.max(threshold) only raises, never lowers, then clamp(0.3,0.9). If base_alpha already exceeds a tier threshold, max() keeps base_alpha.
  - compute_alpha returns base_alpha directly (no clamp) when !adaptive — so a base_alpha outside [0.3,0.9] is returned unclamped in non-adaptive mode (test alpha_non_adaptive_returns_base expects exactly 0.7).
  - boost_bm25_only only boosts when matched_terms is non-empty; a true zero-match item stays at its raw (likely 0.0) score (test fallback_score_no_match_stays_zero asserts exactly 0.0).
  - matched_terms returned from bm25_score are ALREADY alphabetically sorted (query_freq keys sorted) — determinism guarantee. The .take(10)/.take(5) cap then keeps the first N alphabetically.
  - EMAIL_PATTERN literal-pipe bug ([A-Z|a-z]) must be preserved verbatim — it makes '|' a valid TLD char. Do not normalize.
  - EmbeddingScorer.is_available() is false in v0; HybridScorer caches embedding_available at construction so the fallback branch is chosen without re-querying. create_scorer("embedding") therefore always returns Err in v0.
  - Hybrid.is_available() returns true unconditionally (it can always fall back to BM25), independent of embedding availability.

-- algorithm --
  1. CORE FINDING — atomicity is STRUCTURAL, not key-based. Upstream live_zone.rs never reads `tool_use_id` to match a `tool_use` block to a `tool_result` block. There is zero cross-message id-matching code. Instead the design makes splitting/reordering a pair *impossible by construction*. The Go internal/toolpairs helper should encode that invariant, not invent a matcher.
  2. Invariant 1 — message-level immutability of one half: After Phase B PR-B1 retired all message-dropping machinery, compression happens *within* messages, never *between* them. No message is ever dropped, moved, or reordered. Therefore a `tool_use` block (always in an assistant message) and its `tool_result` block (always in the following user message) can never be separated by message-level edits — only the bytes *inside* the latest user message are ever a candidate for mutation.
  3. Invariant 2 — tool_use is hot-zone-excluded (never rewritten): The dispatcher walks blocks of the latest user message and, for each block, reads its `type` (BlockHeader.r#type, default "unknown"). If `HOT_ZONE_BLOCK_TYPES.iter().any(|t| *t == block_type)` it emits SlotKind::HotZone(block_type) and `continue`s — no compressor runs, the block bytes are copied verbatim. HOT_ZONE_BLOCK_TYPES = ["tool_use", "thinking", "redacted_thinking", "compaction"] (exact, ordered, explicit, no prefix matching). The outcome action is BlockAction::Excluded{ reason: HotZoneBlockType }. So the assistant `tool_use` block (and its `id`) is byte-identical on the wire. (In Anthropic shape tool_use blocks live in the assistant message which is *above* the live zone anyway — doubly protected.)
  4. Invariant 3 — tool_result is compressed IN PLACE only, never split from its envelope: For a `tool_result` block the dispatcher does NOT rewrite the whole block. It locates only the inner `content` field's byte range (header.content -> bytes_offset_of within the block) and compresses *only that string value*. The block's `type`, `tool_use_id`, and all sibling keys are outside content_byte_range and are literally copied. So the pairing key `tool_use_id` on the result side is preserved byte-for-byte; the pair link is never even touched.
  5. Invariant 4 — byte-range surgery preserves everything outside edited ranges: out = body[..block_start] || replacement || body[block_end..]. Bytes outside rewritten ranges are literally copied from the input, never re-serialized (preserves SHA-256 of cached prefix/suffix, key order, whitespace, numeric formatting). Replacements are produced in ascending block order and `apply_replacements` sorts defensively before splicing. This guarantees ordering of blocks within a message is preserved too — a result block cannot be reordered past anything.
  6. Dispatcher walk (for context; the toolpairs helper only owns the exclusion test): (1) parse body, get `messages` array; empty -> NoChange. (2) messages_below_frozen_floor = min(frozen_message_count, messages_total). (3) find_latest_user_message_index(messages, floor): iterate messages back-to-front; start = min(floor, len); if offset < start return None; first msg with role=="user" returns its index; else None. None -> NoChange (latest user message is below the frozen floor / in cache hot zone). (4) plan_block_replacements(body_raw, target_idx) -> Vec<PlanSlot>; any shape error degrades to NoChange. (5) for each slot: HotZone -> Excluded; Compressible/StringContent -> compress_one_block. (6) if no block compressed or replacements empty -> NoChange; else apply byte-range surgery -> Modified.
  7. Per-block content extraction inside plan_block_replacements: content==JSON string (legacy shape, content_str.starts_with('"')) -> single StringContent slot (block_index 0). content==array -> per-block: compute byte offset, read BlockHeader; if type in HOT_ZONE_BLOCK_TYPES -> HotZone slot+continue; else by type: `tool_result` -> inner field = header.content; `text` -> inner field = header.text (TextHeader); any other type (image/document/...) -> Compressible with EMPTY content_text and zero-width range (no compressor runs). If the inner field exists but is NOT a JSON string (starts_with('"') is false — i.e. a structured-content array) -> emit Compressible with empty content_text (no-op; structured-array tool_result content is NOT compressed in B3). Else json-unescape the string and emit Compressible with the inner-field byte range (start = block_offset+inner_field_offset, end = start+inner_field_str.len()).
  8. compress_one_block pipeline per Compressible/StringContent slot: (1) Byte-threshold gate — only when content_text non-empty AND len < threshold_for(content_type) -> BlockAction::BelowByteThreshold{content_type, byte_count, threshold_bytes}. Empty content falls through (preserves the 'tool_result with no inner content' pathway). (2) dispatch_compressor(content_text, content_type) -> NoOp -> NoCompressionApplied | Error -> CompressorError | Compressed{strategy, compressed}. (3) CCR marker injection (maybe_inject_ccr_marker) computed BEFORE the token gate so the gate is honest about the ~6-token marker cost. (4) Token-validation rejection: original_tokens=tokenizer.count_text(content_text); compressed_tokens=tokenizer.count_text(compressed_for_replacement); if compressed_tokens >= original_tokens -> RejectedNotSmaller (NOT applied). Else: if CCR store present and a hash was emitted, store.put(hash, original); push Replacement{range: content_byte_range, replacement: json_encode(compressed_for_replacement)}; action Compressed.
  9. OpenAI Chat analog (cross-message pairing reference for the dispatcher API): live zone = latest `role:"tool"` message AND latest `role:"user"` message, handled SEPARATELY (find_latest_role_index for each). Tool messages have shape {role:"tool", tool_call_id, content}. Only the `content` string is compressed in place; `tool_call_id` (the OpenAI pairing key) is outside the edited range and preserved. Earlier tool/user messages are the frozen cache hot zone, never touched. This confirms the universal rule: the pairing id (tool_use_id / tool_call_id) is always preserved because it lives outside the only thing ever rewritten (the inner content string).

-- algorithm --
  1. PHASE 0 — setup: original_len = content.len(); current = content (owned String); total_saved = 0; steps = Vec::new(); cache_keys = Vec::new(). ctx = CompressionContext { query, token_budget }.
  2. PHASE 1 — REFORMATS (serial, registration order). For each reformat transform: (a) compute ratio = current.len() as f64 / original_len.max(1) as f64; (b) EARLY-STOP CHECK AT TOP OF LOOP: if ratio <= config.pipeline.reformat_target_ratio (default 0.5) { break } — i.e. once current has shrunk to <=50% of original, stop reformatting entirely; (c) call transform.apply(&current) -> Result<ReformatOutput, TransformError>; (d) On Ok(out): if out.bytes_saved == 0 { continue } (acceptance gate — discard no-op reformats); else total_saved = total_saved.saturating_add(out.bytes_saved); current = out.output; steps.push(transform.name()). (e) On Err: log and skip (continue) — Internal => WARN, others => TRACE; never propagate.
  3. NOTE on early-stop ordering: the ratio<=target break is evaluated BEFORE applying each transform (top of loop body), so it gates whether the NEXT reformat runs, using the cumulative shrink so far. The FIRST reformat always gets a chance unless input already <=50% (impossible since ratio starts at 1.0).
  4. PHASE 2 — BLOAT ESTIMATION (parallel). Compute reformat_ratio = current.len() as f64 / original_len as f64 (post-reformat size fraction). Then bloat_scores = offloads.par_iter().map(|o| o.estimate_bloat(&current)).collect() — all offload bloat sniffs run in parallel via rayon; each MUST be cheap structural-only, no full pass.
  5. PHASE 3 — OFFLOADS (serial). Iterate offloads.iter().zip(bloat_scores.iter()) as (offload, score). For each: (a) GATE: let above_threshold = *score >= config.pipeline.bloat_threshold (default 0.5); let reformat_underwhelmed = reformat_ratio > config.pipeline.offload_fallback_ratio (default 0.85) && *score > 0.0; if !(above_threshold || reformat_underwhelmed) { continue } — run offload only if bloat score is high enough OR reformats barely helped (output still >85% of original) and there is at least some bloat signal. (b) call offload.apply(&current, ctx, store) -> Result<OffloadOutput, TransformError>. (c) On Ok(out): if out.bytes_saved == 0 { trace-log 'offload accepted but saved zero bytes — discarding'; continue }; else total_saved = total_saved.saturating_add(out.bytes_saved); current = out.output; steps.push(offload.name()); cache_keys.push(out.cache_key). (d) On Err: log+skip as in reformats.
  6. NOTE: reformat_ratio is computed ONCE before the offload loop (using the post-reformat size) and is NOT recomputed as offloads shrink current. So 'reformat_underwhelmed' is judged against the reformat phase result, constant across all offloads.
  7. NOTE: there is NO post-apply compression-ratio gate. The ONLY acceptance check after offload.apply() is bytes_saved==0. There is NO comparison like ratio < transform.min_compression_ratio_for_ccr() — that method/field does not exist in this upstream version.
  8. PHASE 4 — assemble PipelineResult { output: current, bytes_saved: total_saved, steps_applied: steps, cache_keys }. cache_keys contains ONLY offload-emitted keys (reformats contribute none). The orchestrator inserts NO marker text itself; any CCR marker lives inside each OffloadOutput.output produced by the offload's own apply().
